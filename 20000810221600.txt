■讓Linux駕馭世界最快最穩的跑車奔馳

馮文龍/編譯                                                                              
Aug 08 2000

API高速處理器、Linux Networks叢集技術聯手出擊，該叢集技術可以
提供一天 24小時、一年365天不間斷的服務，它同時也能夠提供系統
的負載調節，使得系統瓶頸的發生機率減至最低。

------------------------------------------------------------------------------------------------

API 正致力於開發下一代產品，包括 Alpha 21264-750，以及一個在室
溫下可以跑到1GHz的超級武器 ─ Alpha-21264-1000。

Alpha 750MHz及GegaHz 兩款處理器都是架構在獲得獨家授權的 Alpha 
64 bit技術之上。同時API的Linux策略夥伴也正不斷增加中，包括紅帽
、SuSE、TurboLinux...等。最近一則消息是：Linux  NetworX 已經採用
Alpha的高頻寬記憶體存取及I/O技術，作為其Linux上的叢集技術平台
。Linux NetworX是一家專門研究容錯技術，提供客戶高可用度系統的
公司。兩家公司的合作可以說是讓Linux繼IBM在S390上推出 Linux 版
本後，向高階伺服器技術又向前邁進一大步。

在Linux NetworX的叢集技術產品上執行應用程式，當其中一個處理器
或節點發生故障，工作會自動交給另一個正常的處理緝或節點來處理
。在所謂的肩負警急任務﹙mission critical﹚的系統中，如提供電信服
務、ISP、線上交易系統等，叢集技術可以提供一天24小時、一年365
天不間斷的服務。叢集技術同時也能夠提供系統的負載調節，使得系
統瓶頸的發生機率減至最低。

值得注意的是，當 Linux 的系統技術日趨複雜，以及在其上投注心血
的廠商日益增多，在相容性的問題上，應是所有 Linux 的系統開發廠
商所該有的共識，如何保障使用Linux系統客戶的權益，將是 Linux所
面臨的挑戰。

■更好、更快、更強的 Perl 6

藍雲紋/編譯                                                                              
Aug 08 2000
                                                                                                    
Perl將大幅升級。第六版的 Perl 將支援 Unicode、改善系統記憶體的管
理、加強對XML語法的分析與搜尋。以「更好、更快、也更強」的品
質，與其他網際網路科技並肩合作。

------------------------------------------------------------------------------------------------

有「網路輸送帶」之稱的 Perl，將在往後的 18個月當中，做出大幅度
的升級至第六版。Perl的原作者 Larry Wall在不久前的一次開放原始碼
會議中，指出新版本未來的發展構想。

 Perl 以其連結功能極具彈性而聞名。它可以幫助系統管理者，讓不相
似的系統一起運作。使用 Perl 程式處理的資料，也很容易在各系統間
搬動。而使用 Perl 的網站，也很容易擷取使用者在網頁表格中所填寫
的資料。

Java、C、或 Cobol的設計師在使用前需要定義資料的型態，之後還要
根據定義的型態來使用語言。相對於此， Perl 則屬於比較輕鬆的語言
，更容易與各種型態資料及系統共事。 Perl 擅長於本文處理，將資料
整合並密合在一起。由於Perl開放原始碼，據估計目前約有超過100萬
名的程式設計人員在使用 Perl 工作，每個月約有40萬人次的下載。它
已經成為網路上最普及的一種語言，超過JavaScript或Pythonand Tcl。

不過目前的Perl並不支援 Unicode。新版的 Perl 則將支援，讓中文與日
文在數值指令中，能跟英文一樣。

第六版Perl將對記憶體管理、及分析 XML語法、搜尋 XML標籤的能力
做出改善。以使其與  Java 等其他程式更相容。同時，未來的新版Perl
，還將運用研發模組。將研發主題指定給工作群組，在中央計劃管理
者底下工作。語言研發者可以根據他們過去在第五版的經驗，在第六
版中重寫 Perl 的內部及外部，並且繼續支援現行的主要版本。新版中
並且將包括可以自動編譯第五版程式的工具。

■從Java鑽入Netscape的安全漏洞

謝良奇/編譯                                                                              
Aug 09 2000

Netscape  Navigator及 Netscape Communicator Java直譯器，被發現存有
嚴重的安全漏洞，懷有惡意的Web站台將能透過 Java程式，存取本地
端電腦上的檔案資料，影響範圍包括Netscape Navigator 的所有版本及
Netscape Communicator 4.74版。此外，稍早 TurboLinux 也曾經提出過
Netscape Communicator 4.73的安全問題。

------------------------------------------------------------------------------------------------

知名網路瀏覽器Netscape Navigator及Netscape Communicator的Java直譯
器，被發現存有嚴重的安全問題。懷有惡意的 Web站台能經由該安全
漏洞，透過 Java程式存取本地端電腦，及自本地網路連線中，其它系
統的所有檔案。

這次受到影響的包括 Netscape  Navigator 的所有版本，與 Netscape
Communicator 4.74，甚至波及到那些能夠在 Windows 9x、
Windows NT/2000，及Linux下載執行Java程式的早期版本。而目前仍
處於測試版本的Mozilla及微軟的IE則無此問題。

這個被安全專家視為嚴重的安全漏洞，能允許 Web上的Java程式，在
使用這些 Netscape軟體的電腦上開啟伺服器程序。一旦執行有問題的
Java 程式，懷有惡意的伺服器端將可透過該漏洞，存取系統中的所有
檔案，而像是一般 Windows磁碟分享的本地網路連線系統，也會經由
本地電腦遭受到入侵。

在瀏覽器不會發出任何警告的情況下，懷有惡意的站台或使用者，將
能攫取包括電子郵件、財務資訊甚至是密碼檔案，而該不法的 Java程
式還可以經由傳遞檔案或資訊的方式，將額外的程式碼及網頁散佈到
其它的電腦中。

據媒體報導，該安全漏洞是由 Silicon Valley 公司一名電腦顧問─ Dan 
Brumleve所發現的，但Netscape公司目前為止尚未做任何回應。

被暱稱為 Brown Orifice 的該安全漏洞，其名稱源起為引發問題的 Java
﹙咖啡﹚顏色，與一個數年前出現過的遠端控制程式Back Orifice。安
全專家表示，此次的安全漏洞與Back Orifice極為相似，不同之處在於
，Brown Orifice能透過更多途徑使用，並能自任何網站上運作。

在Netscape 尚未修補該安全漏洞之前，安全專家建議，最好的解決辦
法是將瀏覽器中的Java功能關掉，或是暫時停止使用該瀏覽器，另外
也可以使用有保護功能的防火牆﹙firewall﹚，及入侵偵測系統。

此外，TurboLinux本月2日也曾提出安全性警告：Netscape 
Communicator 4.73及更早的版本在處理JPEG檔案時，會發生緩衝區溢
位的情況，將使遠端Web網站得以在本地端系統上，執行惡意程式碼
或其它行為。

■Linux在中國大陸風起雲湧

黃翔祺/編譯                                                                           
Aug 09 2000

對很多軟體業者而言，中國大陸是目前世界上潛在三個最重要的市場
之一。在大陸到底哪一種系統最受歡迎？答案不是微軟的視窗，也不
是 unix，是軟體界新興寶寶Linux 。因為Linux具有開放原始碼的精神
，且比起其他的系統軟體有更強的價格競爭力。

------------------------------------------------------------------------------------------------

經濟學的市場概念中，人口是決定產品銷售的一項重要因素，如果以
國家來作為市場的區隔，除了已開發國家外，大陸、印度與巴西是目
前世界上潛在最大的三個市場，此三國任何的風吹草動都會對全球產
生決定性的影響。而大陸是這三個潛在市場中，成長最快速、也是人
口最多的國家，大陸的一舉一動自然備受國際矚目。

目前硬體的規格幾乎是透明一致，軟體開發將取決於系統軟體上，因
此，系統軟體可以說在軟體產業中，處於一個關鍵的位置。誰是主流
的系統廠商？在西方世界的個人電腦領域，微軟的 Windows系統幾乎
佔了90% 的市場，在中國是否也意氣風發？答案似乎是否定的。根據
相關資料顯示微軟 Windows系列在中國大陸的地位，目前已經慢慢被 
Linux 所取代。也難怪比爾蓋茲得知中國將採用Linux 為官方的資訊系
統時，會氣得跳腳。

觀察在大陸普遍採用Linux系統的原因，主要有兩個：第一是經濟的因
素；第二則是安全因素。曾有美國資訊專家，利用微軟的系統開後門
，將病毒藏在印表機的驅動程式中，癱瘓敵軍的防衛體系，此事在大
陸軍方引起軒然大波，因為之前大陸的資訊系統幾乎是由微軟建立，
因此安全的顧慮使中國軍事專家喊出：「國家安全來自資訊安全。」
的口號。安全的因素，促使中國利用Linux開放原始碼的特性，由中國
自己進行全面資訊系統中文化的工作。

Linux在中國盛行的因素之二是經濟面，在中國一套微軟系統的價錢約
為一般人一個月薪水的五倍，所以，一般民眾幾乎買不起該產品，微
軟的系統在大陸幾乎都是盜版品。

雖然目前大陸還未正式宣布 Linux為官方作業系統，但市場隨著 Linux 
開發原始碼及價錢便宜的特性，已經在大陸快速延燒，對於其他系統
廠商，如微軟等，有很大的傷害，主要有兩因素：第一是微軟將無法
寡占系統市場，獲取超額的利潤及權利金；此外，隨著Linux盛行，在
亞洲其他國家所引起的骨牌效應，可能才是微軟真正擔心的事吧！

■日本學者利用Beowulf叢集解決方案建構超級電腦

張書銘/編譯                                                                              
Aug 07 2000

日本物理暨化學研究學會四日起開始採用 Beowulf 叢集解決方案，使
用 Linux 為作業軟體，用以模擬並且處理分子力學的問題。這也是日
本第一個採用商業Linux系統的案例。

------------------------------------------------------------------------------------------------

在市場上，處理器的速度平均每十八個月就增加一倍，但是記憶體和
硬碟的速度增加地並沒有CPU快，因此，大多數的應用軟體必須利用
到 cache 以外的記憶體和硬碟來彌補，但是平行化是可以擺脫這些限
制的一種方法。平行計算可以有2倍到500倍速度的提升﹙有時更快﹚
，這種提升是靠單一處理器所辦不到的。 Beowulf 就是一種平行化的
技術。

Beowulf 是目前電腦業界極受注目的一項技術，主要訴求在於擁有整
合並解決大量指令處理的能力，而且也不需要花費太多的預算。
Beowulf 不屬於特定軟體套件、嶄新的網路拓撲或是最新的核心駭客
，而是一套串連 Linux 電腦成超級虛擬平行電腦的技術，任何人也都
可以只靠 Linux 標準套件，不需要額外的輔助就可以建立一套標準的
Beowulf 機器。透過NFS共用/home檔案系統和執行rsh﹙remote  shells﹚
指令，就可以算是擁有一台簡單的雙節點Beowulf機器。 

在Beowulf技術早期的測試中，以 32台﹙32 Nodes﹚Pentium Pro200 建
置的系統便有極驚人的表現，每秒處理約21.9億個浮點運算指令，以
當時的比較而言，超級電腦如 IBM SP2﹙128 Nodes﹚的浮點計算為每
秒 9 5 . 2億個浮點運算指令，在考慮成本效能比後可以瞭解，Beowulf 
System在價格上非常便宜。

日本物理暨化學研究學會為了處理在分子力學上複雜的計算問題，因
此採用了康柏公司的Alpha Beowulf叢集系統。由16台稱為Myrinet的高
性能電腦組成，每台皆配備兩顆Alpha21264A的cpu及1GB的RAM，預
估最高每秒可處理 430 億個浮點運算指令。而同類型的系統於美國也
已經應用於海事氣象局的氣候監測及國家圖書館之中。可預見的是，
Beowulf技術必定將以其低成本高效益的特性吸引更多的使用者。

Aug 10 2000

IBM 每年暑假提供來自美國各地的電腦科學或MBA學生一個暑期實習
的機會，稱為 IBM  Extreme Blue 計劃，目前已經分別在美國東西兩岸
進行，今年美國東岸的舉辦地點在麻州劍橋，西岸則位於矽谷中心，
研究成果相當可觀，使IBM在其Sash科技上取得不凡的應用。

------------------------------------------------------------------------------------------------

IBM推出的暑期實習計劃稱為「極藍」﹙Extreme Blue﹚，提供全美各
地的電腦科學學生或MBA學生一個學習的機會，在此，他們可以有機
會針對新興的科技，例如Linux，寫出新的軟體。

「Weblications」應用軟體，它們的外觀與運作就像標準的電腦程式。
差異只在於以Sash寫成的程式本質上只是內建附加程式邏輯的網頁罷
了。

Sash 的原始版本是微軟視窗版，去年十月在 alphaWorks網站
﹙http://sash.alphaworks.ibm.com﹚上公開。目前 IBM 計畫在8月中對開
放程式碼社群公開Linux版本的Sash，並將張貼在Gnome.org網站上。

其他的兩千年極藍計劃，包括一項針對 Linux 應用軟體的績效監視工
具，以及一項Linux遠端安裝應用軟體。目前為止，IBM尚未宣布這些
新工具的公開日期。

■SourceGear發表開放原始碼的無線通訊入口網站軟體

馮文龍/編譯                               
Aug 07 2000

SourceGear今天宣布一個由其所主導，稱為 lloop的計劃。lloop是一個
建構多重介面社群入口網站的軟體架構，支援諸如WAP手機、PDA，
及桌上型電腦的瀏覽，並且是開放原始碼。

------------------------------------------------------------------------------------------------

lloop是由一組 Python解譯式語言及後端資料庫所組成，它可以產生可
攜式無線通訊設備及桌上型電腦都可閱讀的文件內容。需要公佈的文
件檔案﹙pyp檔案﹚，經由 lloop伺服器處理產生 Python 原始碼，並執
行產生HTML檔案，最後再交由 Apache 網站伺服器發佈送交瀏覽器顯
示。其概念類似微軟的 ASP 或 PHP等伺服器端程序稿語言。lloop可搭
配許多知名的關聯式資料庫引擎，如PostgresSQL、MySQL及 InterBase
等。

在無線通訊設備上，以 Palm VII 為例，lloop 可被下載安裝在Palm上，
做為跟lloop入口網站溝通的介面，而在Palm上的應用程式將會紀錄使
用者曾經到過的lloop入口網站及對應的使用者名稱及密碼。當使用者
在Palm上點選lloop的服務時，Palm上的Clipper 應用程式就會啟動以顯
示在lloop伺服器上的網頁內容。這將使得無線通訊網路繼WAP之後跨
入網際網路的另一跳板，並為打算提供無線通訊網際網路入口服務者
一個最佳解決方案。

lloop是開放原始碼，它的發行可參考GNU的公開授權書，使用者可免
費的取得該軟體，並可依需要自行修改、發行該軟體。

SourceGear 是以開發網際網路上的開發工具及網站的解決方案為專長
的公司，它的專業服務可提供客戶量身定做的網際網路或電子商務解
決方案。

SourceGear 的技術團隊過去均有豐富的網際網路應用程式開發的經驗
，包括了參與微軟早期的IE瀏覽器及與 Motorola 在無線通訊技術的合
作關係，這同時也促成了lloop的誕生。

Slackware 自 1999 年10月發表 7.0 版後，日前再接再厲釋出 7.1版。這
兩個版本間的異動不多，也仍保持Slackware一貫的簡單特色，對於已
經相當熟悉 UNIX 或 Linux 的玩家來說，也是一種選擇。

早期的Linux 玩家對 Slackware 應該不陌生，因為它曾經是唯一可選擇
的 Linux Distribution ﹙套件﹚。隨著 Red Hat、 Mandrake 、 SuSE 等各
家套件的快速發展，以及越來越人性化的圖形安裝／磁碟分割介面設
計，Slackware始終堅守簡單的文字介面設計，使用者必須仔細閱讀安
裝指引，並小心使用磁碟分割程式，才不會發生意外的資料遺失和其
他安裝問題。因此，Slackware較難吸引新的使用者，但舊玩家始終保
持一定的忠誠度。

版本名稱直接從Slackware 4.0跳到 7.X，因為目前Slackware的C Library
已經完全採用glibc程式庫，算是比較進步的版本。Slackware  7.1 發行
的CD有3-4張，亦可從網路上下載3個 iso 檔案，分別可燒成 Install CD
、Source CD 和 Contrib CD。重要改變和內容如下：

·Linux kernel 2.2.16  
·XFree86 3.3.6 ﹙contrib cd 中有較新的 4.0 版﹚
·GNOME 1.2 
·KDE 1.1.2 ﹙contrib cd 中有較新的 1.90 和 1.91 版﹚
·Perl 5.6.0 
·PCMCIA card services 3.1.16 

與 7.0 版不同的地方是，在安裝過程中，Slackware 會自動偵測網路卡
，選擇適當的驅動程式，使用者也可選擇手動調校。值得注意的一點
，在 Contrib CD 中附的許多軟體套件，仍屬於測試版本，如KDE 1.91
和 KOffice pre-Beta1，使用者需小心安裝使用。
