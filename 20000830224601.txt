■Mozilla不等於Netscape！

沈美君/編譯
Aug 28 2000

Mozilla 和 Netscape 在許多使用者的眼中，幾乎是同義字。事實上
，兩者之間有不少差異處，近來在 mozilla.general.newsgroup有熱
烈討論。要釐清這個問題，可以由兩個層面著手：Mozilla/Netscape 
是什麼？而 Netscape 和 mozilla.org 的相互關係又是什麼？

------

最近在  mozilla.general.newsgroup 新聞討論群組上有一連串有趣
的文章，標題為「Netscape == Mozilla 」，主要討論這兩個一向被
大部分使用者視為同義詞的問題。

Mozilla 不等於 Netscape。這對 mozilla.org 發展社群的程式設計
師來說，是很容易瞭解的道理；不過對於一般使用者而言，要瞭解這
些社群、軟體套件、發展計畫、公司組織之間的關係，似乎絕非必要
也很難知道它們之間的差異。

Netscape Communications Corporation ﹙以下簡稱網景﹚是以發展
瀏覽器起家的網路公司，他們所研發的瀏覽器  Netscape Navigator
可以說是定義了瀏覽器的產業標準，1990年代後期，Microsoft、Sun 
和 Apple 依據這樣的標準，開始發展各自的瀏覽器。

網景投入資金支援發展mozilla.org的計劃。 mozilla.org 是一個致
力於開發 Mozilla Porject 的發展者社群，而 Mozilla Project 是
一個開放原始碼發展計畫，由 Mozilla software components﹙包括
browser、email client、chat client ﹚和 Mozilla Technologies
﹙包含 XUL 及 XPCOM 技術﹚所組成的一個發展平台。

Mozilla 的 XUL 技術﹙Extensible User Interface，可擴充性的使
用者介面﹚，允許發展者使用單一程式語言，就可以建立適合各種作
業系統平台的瀏覽器使用介面，而不需重新撰寫程式碼。XPCOM 技術
﹙Cross Platform Component Object Model ，跨平台物件模性﹚讓
程式設計師所撰寫的軟體原件，能符合各種平台的需求，不需針對每
種平台再重新撰寫原件。

Mozilla這兩種技術，相較於微軟的 IE5.5 的優勢，就是跨平台這項
特性。網景的  Netscape 6 和 Netscape 7 採用 Mozilla 瀏覽器引
擎，發展者針對採 Mozilla技術所撰寫的網站應用程式，幾乎能在所
有的作業平台上執行。反觀 IE ，同一個系統專案必須針對不同平台
，再重新發展各自的套件；此外， Mozilla 幾乎 100% 符合 W3C 所
提出的標準，提供發展者最大的支援。

雖然網景是背後支持 Mozilla project  的公司組織，但其實讓此計
畫能順利進行的，是靠來自世界各地的程式設計師，透過發展者社群
以及 mailing list，做研發討論、提供各種 patches、bug fixes，
這種知識分享的機制和開放的發展精神，即使網景不再繼續提供支援
，Mozilla project 也能發展下去。

■第一台網路數位相機─Axis 2100

張書銘/編譯                                      
Aug 28 2000

第一台擁有網路功能的數位相機終於問世，Axis擁有數位相機及攝影
機的功能，提供使用者拍攝640x480或320x240像素的的照片，或拍攝
每秒跑3到10格畫面的影片。另外，還可使用FTP遠端搖控影像上傳動
作，也就是說，Axis 2100 也可做為視訊會議設備。

------

在勾勒未來化住宅時，使任何一項家電用品都可以上網，已經是一種
基礎概念。而現在電腦周邊設備本也紛紛發展網路功能。 Axis 2100
即是在這種觀念下設計出來的一台網路數位相機。

Axis 2100擁有數位相機及攝影機的功能，提供使用者拍攝640x480或
320x240相素的的相片，或拍攝每秒跑3到10格畫面的影片。如果使用
傳統的數位相機，想把影像透過網路展示，必須把影像轉到電腦中，
再連接網路才行。

而現在Axis 2100內建有Linux系統，及自己的網路伺服器，也有一組
乙太網路的連接插座，所以使用者可以利用瀏覽器來觀賞 Axis 2100
所拍的照片。當然，配備有Linux的Axis 2100也提供了安全控管的機
制，和一般使用Linux 的系統一樣，可以設定使用者以及其權限，保
障在網路上的安全性，防止其他網路使用者登入內部的危險。

另外，只要為它設定好IP位置，也可以使用 FTP來遠端搖控影像上傳
的動作。也就是說，Axis 2100 也可以做為視訊會議的設備。同時也
可以在Axis的開發部網頁找到所需的編譯器和函式庫，來為Axis撰寫
一些用於相機的特殊的程式，讓使用者可以創造一台符合自己需求的
Axis 2100。

■Caldera併購SCO後首先面臨的難題─授權金

黃翔祺/編譯                             
Aug 28 2000

前一陣子，受到市場人士及分析師注目的Caldera與SCO併購案，最近
因為權利金的問題，Caldera與SCO間產生許多爭議。同時， Caldera
在面對開放原始碼時，也一改以往完全支持的態度，逐漸轉向保守。

------

日前，Linux銷售商 Caldera  Systems 收購 Santa Cruz Operation
﹙SCO﹚的Unix系列產品，並積極整合過去Unix與Linux，希望能做出
第一個Unix/Linux通用的平台。原本分析師都認為Unix還保有若干優
勢，但在Caldera收購SCO的產品之後，象徵Unix時代的離去，而脫胎
自Unix的Linux成為耀眼新星，且市場人士還信心滿滿地認為「linux
已經成為未來最受注目的系統軟體」。不過，現在卻因為Caldera 該
付多少權利金而引起爭議。

整個事件的起因，是SCO所掌握的Unix程式碼來源太過於複雜，SCO曾
經受過許多不同公司的投資，在期間進行多次的改版動作，其版權問
題相當複雜，Caldera 為了要不要公開原始碼的問題傷透腦筋。若要
公開原始碼，過去的股東，包括微軟、網威、昇陽、AT&amp;T等公司，恐
怕都會要求鉅額的權利金。若不公開原始碼，而將Unix與 Linux融合
在一起，可能會被死忠開放原始碼的Linux社群反彈。

Caldera總裁Ransom Love為了這件事傷透腦筋，在華爾街面對財務分
析師時，他不得不說：「我們認為開放原始碼並不是最好的選擇，畢
竟公司還是要有競爭力。」而面對過去並肩作戰的開放原始碼社群，
則宣稱：「我們要創新，回饋大眾。」因此，Ransom  Love被譏為雙
面人。

目前，整件事還在持續發酵中，市場人士認為，Caldera 目前對於支
付權利金有三種方式：第一，Caldera 並不擁有該智慧財產權，也不
能使用。這樣的模式似乎是行不通的，畢竟Caldera就是希望取得SCO
在Unix上的智慧財產權。第二，Caldera 有第一優先順位進行公開，
但限於經過編碼的程式。第三，公開SCO過去Unix的軟體 Unixware，
但由於Unix過去與許多系統之間的關連性，使得這方式也變得大不可
能實行。

除此之外，還有一段意外的插曲，Unix的創始人Thompson公開在媒體
上批評Linux 終究會失敗，他認為開放原始碼雖然使許多人能接近程
式、修改程式，但由於程式設計人員的素質良莠不齊，一段一段的程
式碼，素質也不同，這使得整個Linux 系統變得相當不穩定。同時，
也增加開發程式公司的問題。雖然這看起來只是一段意外的插曲，但
似乎也反映了過去IT產業的程式設計人員，在面對Linux 系統時的心
態。

Caldera目前正面臨嚴酷的考驗，如果Caldera考慮公司本身的利潤時
，可能必須要放棄開放原始碼的作法，但如果放棄開放原始碼，會不
會間接影響到開放原始碼的發展風氣？這是Linux 支持者所擔心的問
題。

■Linux邁向IC產業

馮文龍/編譯              
Aug 29 2000

Mentor Graphics 公司宣佈，已經成功將原本執行在UNIX上的IC工作
站移植到Linux。此Linux版IC工作站是採用RedHat 6.0架構的，並且
在價格與功能上與UNIX版的IC工作站一致，同時表現上也非常穩定。

------

Mentor Graphics的Linux版IC工作站使用RedHat 6.0架構，Mentor的
發言人強調，它與UNIX版的IC工作站的價格與功能完全一致，並且表
現上非常穩定。

IC工作站基本上是由許多的工具整合而成，例如Schematic Capture
、schematic driven layout、一組參數化的設備產生器、交談式的
規則驗證工具、一個混合的cell-and-block配置及路徑工具以及工單
變更等功能。它讓設計與輸出成形可以同步進行，並使得IC設計的生
產力10倍於傳統的設計方式。

Mentor將原本只在UNIX上執行的IC工作站延伸到Linux 上，有兩種基
本的市場需求，即大型半導體廠商提供給非現場工作設計師的工作環
境，以及小型無實驗室廠商需求的低成本工具。過去這些需求小型、
低成本的廠商大多選擇一些免費或低成本的工具軟體上，但最後都發
覺這些低成本工具軟體的整合性功能難以滿足需求，於是不得不更換
為整合性較佳、高效能的工具軟體，而Mentor Graphics的Linux版IC
工作站則正好符合他們的需求。

Mentor在移植IC工作站到Linux 的過程非常順利，並表示：「因為它
只是另一個UNIX嘛！」。但是Mentor的IC事業部協理同時也建議，在
PC環境執行IC工作站的最佳硬體環境需求為：盡量使用所能買到的最
快CPU及最大記憶體，他並且強調，Mentor都在擁有500MB以上記憶體
的環境中執行。

■Jabber─嵌入式Linux、XML及即時傳訊 三者結合的新巨浪

謝良奇/編譯
Aug 28 2000

第一套開放原始碼的即時訊息傳送平台 Jabber，將在嵌入式Linux、
XML 及即時傳訊系統間展開對話疆域，作為各類型裝置間，訊息傳送
的基礎架構，更有可能改變網際網路的面貌。

------

嵌入式Linux系統結合即時傳訊﹙Instant Messaging﹚，以及隨之湧
現的各類新應用軟體及服務，將為我們帶來思考，使用網際網路的新
方式，第一套開放原始碼的即時訊息傳送平台Jabber，是一波由嵌入
式Linux、XML及即時傳訊，三者結合的新巨浪。

Linux 系統近來逐漸在行動設備市場受到矚目，該類設備通常是在嵌
入式或即時作業系統上運作，雖然 Linux並非即時作業系統，但因為
其小巧及容易自訂化的模組特性，最重要的是其強大的網路原生能力
，相當適合網路連接需求漸增的嵌入式市場。

為了補足 Linux系統在即時能力的缺陷，一套即時傳訊系統是必須的
，Jabber即是第一套開放原始碼的即時訊息傳送平台，盡管早已有了
AOL的 AIM及ICQ等相似系統，但這些系統不是開放原始碼，不相容的
傳訊系統間，也無法互相進行通訊。Jabber的創造者Jeremie Miller
，就是為了解決不同即時傳訊系統間的通訊問題，而設計出Jabber的
雛形。

Jabber 採用主從式架構，伺服器與客戶端之間，透過XML的資料流交
談，支援如telnet連線、AIM、ICQ及其它客戶端軟體。Jabber的通訊
協定由 XML封包組成，包含訊息、狀態、查詢或資訊等三種基本類型
。而每一架伺服器之間的通訊，只有單純的XML資料流的路由動作。

XML 是一種標示語言，能用來包裝及交換結構性資料，讓傳輸雙方瞭
解通訊的資料格式，是 Jabber中極為重要的元素。透過XML組織資料
的功能，Jabber不僅能讓人們相互傳訊，更進一步能付與人與機器，
機器與機器間相互通訊的能力。未來嵌入式應用軟體將能利用Jabber
作為訊息傳送的基礎架構。

Jeremie的願景是， Jabber能夠扮演 XML路由器的角色，經由嵌入式
Linux，及各類型支援XML的作業系統，變得無所不在，進而擴展網際
網路的基礎架構，以容納各類型不同設備間，更為廣泛的動態即時關
係及相互運作。

■Linux的桌面使用環境：還有一段路要走

沈美君/編譯
Aug 29 2000

各家 Linux 廠商為了開發 EndUser 這塊市場大餅，無不卯足全力研
發更容易上手的 GUI﹙圖形使用者介面﹚桌面使用環境，亦極力爭取
其他廠家合作，來發展各種符合一般消費者需要的應用軟體。雖然軟
硬體支援幾乎可以支援一般企業的作業，但是對普通沒有技術背景的
使用者來說，還有很大的發展空間。

------

對於具有技術背景的電腦使用者來說，使用具穩定及安全性的 Linux 
作為作業系統，絕對有千百個好過用 Windows  的理由。但對一般使
用者而言，作業系統簡單容易上手、以及有他們需要的應用軟體，是
最重要的兩件事情。

微軟以 Windows 系列，佔得目前 90% 以上的 Desktop user 作業系
統市場，除了是運用商業及市場行銷策略之外，易於使用也是極大利
基之一。在前幾年，一般消費者若無一定專業知識，實在沒辦法能將 
Linux  當成平台來使用，而能裝在 Linux 上的應用軟體也屈指可數
。不過近年來由於各家 Open Source 廠商策略聯盟，積極開發GUI使
用者介面，並尋求各種應用軟體的發展，在硬體上的支援也明顯完整
了許多。

不過，若我們由桌面整合、應用軟體、系統工具、辦公室套件、支援
文件、列印等幾個面向來討論，可以發現：雖然 Linux的桌面整合環
境已經漸趨完備，不過還有很大的發展空間。

目前的主流 wfm﹙圖形介面視窗管理員﹚為 KDE與 GNOME，各有屬於
自己的發展計畫及支持者，兩者亦有各自的軟體套件組合。但是若在
使用 KDE 的情況下，欲安裝 GNOME 下的應用軟體，則系統會自動將
必須的 GTk 和 GNOME libraries 安裝進去，但 KDE 和 GNOME 的應
用軟體間，並沒有辦法使用 Drag&amp;Drop﹙拖放﹚的功能。

在應用軟體的多媒體方面，雖然已經有可以播放影音的 Linux軟體，
許多遊戲還是只能在 Windows  下跑，大部分的教育自學軟體也沒有 
for Linux 的版本。幸好在辦公室套件方面，已經有 StarOffice 或
Word Perfect，足以提供媲美 MS Office  的試算表、文字處理、簡
報等功能，但是目前尚未有中文版產生。

不過，系統工具方面的磁碟分割，對一般使用者來說還是很頭痛，雖
然 Mandrake、RedHat 等廠商已經將磁碟分割改為圖形介面，也有自
動分割安裝模式，但是若能再簡單一些，對沒有技術背景的使用者來
說，會更容易上手。在 X window 的設定方面，目前已經有自動偵測
顯示卡晶片和螢幕解析度的自動調教，減少使用者設定錯誤的麻煩。

不過在列印方面就沒這麼幸運了，目前主要還是透過 ghostscript，
但如此一來，有些印表機支援的功能就無法使用。另一個沒有整合完

、HTML、 on line help ﹚，或者根本沒有說明文件，不像 Windows
一樣有統一的 Help 格式。

Linux 的支援漸多，對企業及公司的使用上來說，雖然已經漸趨完備
，但對一般家用使用者而言，還有一段不短的路要走。

■英特爾共通資料安全性結構CDSA 2.0出爐

藍雲紋/編譯                                
Aug 28 2000

專為個人電腦、工作站、和伺服器設計的資料安全下部組織CDSA 2.0
出爐，可以隨插即用，讓軟體研發者整合容易，也讓資訊科技管理者
透過網際操作分享資料資源的安全性無虞。

------

共通資料安全性結構 CDSA﹙Common Data Security Architecture﹚
是由英特爾為所提供並開放原始碼的一種新的強大功能。

CDSA以數字認證及可移動數字記號兩項資料安全為基礎，共分四層結
構，底層為服務提供模組，提供加密原理、基礎認證操作設備、與儲
存，支援ANSI C和 C++等語言，讓結構既模組化又可擴充，鼓勵獨立
軟體代理商研發新功能及改善績效，提高競爭力。五種整合到CDSA的
基本服務提供模組類型，分別為：加密服務提供模組、信任政策模組
、認證圖書館模組、和授權計算模組。

在CDSA 2.0版中，最關鍵的元件為「可選擇模組管理員"﹙ Elective
Module Manager﹚，代理商可以用EMM來新增生物檢測授權。

CDSA被認為能保護電子商務交易及私人通訊，原因有以下幾點：它能
提供消費者下一代安全服務、可在不同平台上工作、及使用各種網路
研發語言。它讓研發者能用自己最專長的工具，建立它們的產品和服
務。也由於 CDSA 已經解決了企業安全中最棘手的問題了，研發者便
能集中精神在提昇自己的產品本身，縮短產品上市時間。

專家相信：安全軟體的原始碼一經開放，也必將加強軟體安全性的強
度。對資訊科技管理者來說，CDSA沒有隱藏的弱點或不知道的功能。
對軟體研發者而言，標準化介面容易與其他安全區塊整合。至於從消
費者的角度來說，則 CDSA 的模組化就像隨插隨用的安全聰明卡。

它提供通過效能測試的網路可操作性，加速工業在提供安全提昇的解
決方案，這方面的能力。提供銀行及信用卡客戶更佳的保護與方便性
。它的主要受益對象將是電子商務業者。

■eLinux與TeamLinux協力提供專業服務

蕭喜那/編譯
Aug 25 2000

eLinux經由其安全的採購網站，提供Linux社群一次完成的企業解決方
案，TeamLinux供應企業專業的Linux科技解決方案，這兩家公司近日
宣布將組成策略聯盟，以提供Linux社群更完整與專業的企業解決方
案。
 
------

eLinux與TeamLinux近日共同宣布一項提供專業服務的合作案，從此次
的合作案中，eLinux目前所銷售的產品，以及所提供的定製化安裝與
支援服務，品質都將因此而獲得提升。同時，TeamLinux所具有的開
放原始碼與Linux服務的能力也可以發揮影響力。


這項合作案藉由TeamLinux所提供的關鍵性企業層級解決方案，將可
以補足eLinux現行的基礎架構以及技術服務。此次兩家公司之所以結
盟，原因很簡單，因為eLinux早就將重心擺在IT顧客上，提供它們完
整的設計、採購、安裝、與管理等具有相互可操作性的解決方案，並
且設立單一的聯絡點終端（也就是網站）以提供這些服務。

eLinux的母公司是IdeaMall，是NASDAQ公開上市的公司，一九九九年
的銷售額是七億三千二百萬美金。IdealMall設立eLinux.com，其目標即
在於提供一次解決的購物、支援、與社群服務資源，給Linux社群所用
。六月出版的Linux Magazine把eLinux.com列為全球資訊網上的百大購物
網站。eLinux可以為多賣方、多平台的企業解決方案，個別安裝伺服器
、桌上型電腦、以及筆記型電腦。eLinux.com的網站上尚包括技術支援
、新聞、討論區以及依主題劃分的文章。

TeamLinux提供由Linux、開放原始碼、以及其他科技所整合而成的解決
方案，包括企業專業化服務、訓練與產品。該公司所提供的服務包括
：企業評估、網路設計與管理、資訊安全、資料庫設計、網站設計與
架設、開放原始碼與Linux訓練等。

TeamLinux的董事長兼執行長Dan Maher認為與eLinux的合作呼應了該公
司的成長策略，亦即積極加速與Linux及開放原始碼的業界領導廠商組
成策略性結盟。eLinux的執行長Dan De Vries則說與TeamLinux的結盟
對於該公司提供完整的科技解決方案，給中小企業市場、政府市場、
以及教育市場，有相當大的幫助。
