■KDE與GNOME的大對決

張書銘/編譯                                      
Aug 21 2000

提供輕鬆上手的圖形化操作界面，是吸引使用者採用一套系統的最好
手段之一。在Linux提供這種服務的 KDE以及GNOME，最近各自推動著
新的開發及合作案，企圖在這場競爭中甩開對方，以成為使用者大眾
統一的標準。可預期的是這樣的競爭，必然可以使得Linux 丟掉操作
界面不夠友善的沉重包袱。

------

GNOME﹙GNU Network Object Model Environment﹚是屬於GNU計劃中
的一部份。GNU計劃開始於1984年，專注於發展類似 UNIX且完全免費
的作業系統。而 GNOME則以提供讓使用者容易操作和設定電腦環境的
工具為目標。同樣的，KDE 也致力於提供相同的服務。而在這種服務
領域重疊的情況下，兩套軟體向來都被Linux 使用者當做比較的對象
。因此，每當兩套軟體有所動作時，總是會引起熱烈的討論。

在最近的一次開發案中，系統供應商的大廠昇陽以及惠普公司表示將
把GNOME 做為它們系統的預設桌面環境。而昇陽更是投入了大量的研
發工程師開發並改良GNOME。對於這項大廠支持GNOME的舉動，有不少
輿論都表示GNOME 強調可以多種程式語言來撰寫，並不受限於單一語
言，且任一元件的開發或修改均不受限於某家廠商的特性，是它能夠
獲得青睞的主要原因。

面對業界龍頭選擇了GNOME，KDE的開發團隊則表示並不影響它們的生
存空間。 KDE團隊在得知該項計劃後，發布新聞表示兩套軟體都已經
在使用者大眾間有了固定的支持比率，即使 GNOME有昇陽和惠普的行
銷和開發支援，KDE仍然不會受到影響。而KDE團隊更直接的表示，大
量投入研發人員並不能保證一項研發計劃的成功。言下之意是對於昇
陽試圖挑戰KDE地位的舉動完全不值一提。

而KDE也有相對應的合作案做為反制。Corel公司21日宣佈與兩個開發
開放原始碼的主要社群合作開發Linux產品。主要的內容，是Corel及
KDE  Project 將攜手增強KDE GUI的功能，並將最後研發成果應用到
Corel Linux 之上。根據Corel公司表示，這套新的Linux安裝及圖形
使用者介面將於今秋推出。

■Solaris 8與視窗2000互別苗頭，激戰即將展開

藍雲紋/編譯 
Aug 22 2000

昇陽的Solaris 8 提出暖修復、新增功能、叢集服務升級、頻寬控制
、安全性功能、採納開放原始碼等多項新特色較勁。微軟則將推出視
窗 2000 Datacenter 與其廝殺。

------

昇陽表示，自從開始提供免費作業環境以來，半個月內已經出貨 7萬
套的 Solaris 8。而微軟方面也不甘示弱地說，自從視窗2000發行以
來，一個半月的時間，他們則已經銷售了100 萬套的專業、伺服器與
進階伺服器版，而其中90%都在桌上型電腦上。

為減低關機時間，Sloaris 8 允許在舊版仍在執行的狀態下，升級的
方法。並在採購硬體之前，對軟體更新所可能帶來的當機，先行偵測
分析。在新增功能方面，Solaris 8 將過去昂貴的套件整合到作業系
統中，以節省許多撰寫 Perl 的時間。另外來新增了 Admin Pack 來
改善第三方產品和多重作業系統環境的管理，以支援網路基礎的企業
管理標準。叢集服務也保證將提出3.0 版。

在頻寬方面，管理者可以將企業內網路交通分級，來控制使用者頻寬
。還可對服務設置頻寬優先權和保證。安全性方面，也提供了些方法
。Solaris 8 也支援許多開放原始碼的應用，例如  Apache 1.3.9。
Linux也在受支援的行列中。唯一的遺憾是並未支援Java。

昇陽表示，昇陽並不是像微軟一樣，靠著向電腦製造商索取作業系統
的執照權而賺錢，昇陽 大部分的獲利來自以 Solaris 為主的大企業
的硬體和系統的採購。微軟想要將他們的食物鏈，從桌上型電腦轉向
資料中心，甚至他們向來所專長的中級伺服器。不過昇陽表示，他們
已經注意到了。

有些分析師認為，真正的激戰要到今年稍晚的秋、冬季，等視窗2000
Datacenter伺服器上市才會開始。視窗2000  Datacenter伺服器直接
鎖定大量的Unix商業客戶。可運行四節點叢集、64GB記憶體、和進階
叢集服務，例如程序控制管理。微軟說，我們非常高興顧客採用我們
的產品，來研發他們的B2B平台和電子商務平台。Datacenter 所帶來
的四節點叢集合超大量資料庫交易，一定能讓客戶感受到進階伺服器
的力量；不過，我們不會等到 Datacenter 版出來，才去搶攻昇陽的
核心市場。

■微軟否認移植應用軟體到Linux/Unix

黃翔祺/編譯                             
Aug 21 2000

是什麼原因，讓微軟對於媒體所披露的「微軟將移植他的視窗應用軟
體至 linux平台」的訊息，如此憂心？就產品來說，雖然微軟想要提
供linux版的相關office 軟體，但是就目前來說，該產品部門並不會
得到長期的實質利益。就公司層次來說，這樣的舉動將會危害過去微
軟所建立的軟體王國。

------

過去兩星期中，微軟忙著澄清幾家媒體所披露的訊息：「微軟將移植
他的視窗應用軟體由視窗平台至linux 平台」，令微軟緊張的是，其
中一家媒體 Winfo一向與微軟公司密切，是專門提供微軟公司產品資
訊網站，也流傳這樣的資訊，更是雪上加霜一般。

市場流言指出，微軟將透過與MainSoft的服務，來進行移植其應用軟
體至linux平台。究竟 Mainsoft是什麼樣的公司？為什麼微軟對自己
產品在軟體上會這麼緊張呢？

MainSoft成立於1991年，主要的業務是將微軟所生產的軟體，利用自
己所研發的 MainWin，將微軟所提供的原始碼，轉譯為在 Unix 可以
使用的應用軟體。目前，Mainsoft正在將微軟的瀏覽器軟體及媒體播
放程式移植至 Unix 平台，但是微軟與 MainSoft 皆正式否認其他的
Office 應用軟體及信件應用軟體， Outlook會移植至Unix或linux平
台。

為什麼微軟在 linux市場佔有率節節上升時，不提供其相關應用軟體
呢？

就產品部門層次來說，提供linux 版的應用軟體，目前，並不會得到
很大的效益，原因是開放原始碼之後，將使更多人利用該原始碼，並
改寫其軟體，這樣將危害到微軟長期的利益。雖然在 linux上不全然
要求提供原始碼，但是由於昇陽公司所研發之Staroffice接近上市階
段，最後勢必逼迫微軟開放其原始碼，壓縮產品所獲得的利益。

就公司策略層次來說，微軟一向採取搭配合作策略，讓視窗與其應用
軟體部門互相利用，達到實質獨佔整個市場的效果。讓其應用軟體部
門提供原始碼給linux社群，對於Windows來說，不啻是一大損失，就
公司來說是不允許的。

目前，短期內消費者應該看不到微軟提供 Unix或linux等相關應用軟
體出現，除非美國司法部宣判微軟應該將其Windows 及軟體應用部門
分別獨立，而軟體應用部門迫於市場因素推出linux版的office 及相
關軟體。

■未上市先轟動 ─ 64位元Itanium晶片

藍雲紋/編譯
Aug 24 2000                                 

紅帽為它提供服務、SeSE為它提供免費測試環境、惠普發表模擬設備
、TurboLinux等的爭先恐後，讓IA-64系列晶片之一的Itanium成為市
場新寵。

------

英特爾的Itanium晶片，都還沒上市，就已經為Linux相關公司紅帽創
造利潤了。 Itanium是紅帽公司64位元晶片系列的首批產品，因為英
特爾的延遲，要等到明年才能到貨。不過，紅帽公司已經宣佈將提供
JumpSmart服務，給想將軟硬體轉向Itanium的公司。服務的內容包括
：研發者可以透過網際網路使用一群IA-64電腦、對於在IA-64上安裝
組態 Linux提供訓練、協助將軟體轉換到新結構、紅帽的軟硬體認證
、和市場行銷協助。

英特爾相信，在 IA-64晶片系列的競爭中，將首先取得優勢的三家作
業系統，包括 Linux 、 Windows 64 、和 IBM AIX 5L 版的 Unix。
三月份，TurboLinux針對IA-64提供一版Linux。紅帽，五月緊跟在後
。至此，對手SuSE和Caldera也將成形。

1992年成立的SuSE是一家開放原始碼作業系統軟體的解決方案提供公
司。總部設在德國，以美國加州為基地的 SuSE Linux AG，全心投入
支援Linux社群，及開放原始碼的研發。SuSE 日前宣布，將提供免費
測試環境給新的64位元結構，讓研發者可以透過網路，將應用程式轉
到IA-64平台及其首要處理器Itanium上。

Itanium的吸引力如此叫人眼紅，目前，除了英特爾的競爭對手AMD投
入設計Sledgehammer晶片，希望與 IA-64系列競爭外；微軟也已經送
出即將上市的64位元視窗拷貝，給擁有 IA-64機器的客戶。而原先以
設計 IA-64晶片為主的惠普公司，也將繼續支援這項努力，並已發表
了一種模擬設備，供測試軟體實驗是否可在IA-64原型上執行。

■新的開放原始碼瀏覽器K-Meleon

蕭喜那/編譯 
Aug 25 2000

K-Meleon是由 Christophe Thibault所開發的開放原始碼瀏覽器，它
的外觀看起來像是微軟的IE ，但是它內部使用的是Mozilla所開發的
翻譯引擎Gecko。K-Meleo結合了IE下載比較快的介面，以及Gecko 加
速網頁展示的能力。

------ 

以開發 K-Jofol ﹙視窗作業系統下的多媒體音訊播放軟體﹚ 聞名的
Christophe Thibault ，近日公開一個非常單純的全球資訊網瀏覽器
，稱為「K-Meleon」，它的外觀看起來很像微軟的IE，但是它的內部
使用的是Mozilla的Gecko翻譯引擎﹙rendering engine﹚。K-Meleon
是一個新的開放程式碼瀏覽器。

K-Meleon採用了IE的很多特點，包括我的最愛以及幾乎完全相同的工
具列。K-Meleon就像Opera瀏覽器以及Galeon 一樣，最大的特色在於
所佔的空間很小並且下載時間很短，完整的安裝只要2.85MB，如果電
腦已經安裝了M17 Mozilla 測試版的話，可以安裝最精簡的版本，只
要183KB。

另外也很重要的一點是，K-Meleon 是首先使用Gecko翻譯引擎的應用
軟體之一，Gecko是由Mozilla.org的小組所開發的，它可以加速網頁
展示的速度。預定於年底公開的Netscape 6將使用Gecko， Netscape
的母公司─美國線上─也正鼓勵開發人員在其他應用軟體與產品內使
用Gecko。

為了不被別人領先，微軟也正發展自己的翻譯引擎，稱為 Universal
Canvas，將成為該公司下一代網路使用者經驗中關鍵的一部分。不過
，微軟尚未回應對別的產品可能抄襲IE的外觀與感覺有何看法。

不過，K-Meleon的這個測試版仍有許多不盡完美之處，包括背景的選
單、HTTPS、歷史、籤餅的儲存、以及MIME等。Christophe 已經將完
整的原始碼公開，他希望有人可以拿這個原始碼去做些改進。有興趣
的開發人員可以依據GNU授權規定取得K-Meleon。

■好萊塢電影工業贏得DVD破解案

謝良奇/編譯                                   
Aug 21 2000

美國聯邦地院傳來令人失望的消息：數家電影製片公司贏得了 DVD破
解訴訟案。美國聯邦地方法院17日作出判決：任何人不得繼續在網站
上提供能在Linux上，對DVD影片進行解碼及播放的DeCSS軟體。

------

美國聯邦地方法院在本月17日，對 DVD破解訴訟案作出判決，判決結
果導向支持發起訴訟的數家電影製片公司，禁止著名的駭客雜誌及網
站─2600的出版者Eric Corley，在網站上散布能在Linux系統上，對
DVD影片解碼及播放的軟體─DeCSS。

程式原始碼是一種電腦指令的表達法，由於一向被視為一種言論方式
，被認為擁有言論自由保護，然而，美國地方法官Lewis Kaplan在其
支持電影工業的判決中認為，程式碼的的使用及散布，必須要受到社
會的管理。

在上個月的審判中，Lewis Kaplan否決了辯方律師認為，電腦程式碼
為自由表達的說法，認為2600散布此電腦程式碼，是毫無根據的行為
，並且不將程式碼視為單純的表達法看待。而該雜誌則提出爭辯，說
明DeCSS的散布行為，的確受到第一修正案的言論自由保護。

法官 Lewis Kaplan 針對2600的93頁判決書，可以視為是好萊塢最近
為了試圖阻止數位錄影的著作權侵害，其中的一項動作成果。美國電
影協會主席即在聲明中表示，該判決對消費者及正當科技來說，是一
項勝利。

然而深刻考慮過Kaplan 的此項判決，如果公開破解DVD加密方法的工
具，屬於非法行為，那麼未來針對任何一種加密演算法，提出批評以
說明其弱點，是否也將屬非法。

■LinuxWorld群雄聚集，新版Linux核心秋天推出

張書銘/編譯                                    
Aug 21 2000

托瓦茲﹙Linus Torvalds﹚在年度 LinuxWorld 博覽會中表示，新版
Linux核心作業系統即將推出。儘管目前各種Unix及Linux延伸出來的
作業系統競爭激烈，市場佔有率數一數二的托瓦茲卻信心滿滿的表示
，配置最新核心的版本將會如期在今年秋天推出。

------

美國一年一度的LinuxWorld博覽會今年可謂盛況空前，無論是參展或
是參觀者的數目都比過去提高許多。而在今年的展覽中，多家參展團
體及廠商更是提出了最新的開發案及產品，令人目不暇給。堪稱為近
年來Linux產業界最大盛事。

LinuxWorld博覽會對業界來說，無疑的是打響自己名號的一條捷徑，
也因此相關於這項產業的各種團體，從開發者、系統供應商一直到雜
誌以及出版業者無不努力的對展覽提供各方面的幫助。今年也不例外
，在各個大公司以及開發團隊熱烈的參與，試圖在展覽中為自己營造
出最大聲勢的期望之下。在今年的展覽中，照例有許多有名的公司提
出各式各樣的策劃來吸引大眾的注目。

首先是 IDG 以及托瓦茲提供了25000美元的獎金給開發、促進 Linux
的社群團體，獎勵他們在研發及推行上所做的努力。今年，得到本獎
項的是Debian。這個團體在近年來，提供了不少Linux 的資訊給使用
者大眾，對於促進使用Linux 風潮有相當大的助益。另外，在網路上
最著名的網站 LinuxWorld 則票選出了年度最佳Linux出版品為Linux
Journal。

在各公司暨開發團體的展覽中，則以以下所提出的消息最令人注目，
一是GNOME開發協會訂出了新的GNOME標準。對於開發易學易用的界面
向來不遺餘力的GNOME 協會在提出新的標準後，其他的開發者多半也
必須跟著新的標準走。而最近HP及Sun宣佈將把GNOME做為新系統的預
設桌面環境，並且投入大量人員做開發工作，屆時GNOME 會有怎樣的
變化值得期待。

另外，托瓦茲則在會中表示他們新的Linux 核心即將推出，儘管目前
各種Unix及Linux 延伸出來的作業系統競爭激烈，市場佔有率數一數
二的托瓦茲卻信心滿滿的表示，配置最新核心的版本將會如期在今年
秋季推出。

■紅帽與Amiga合作開發Linux平台多媒體應用

沈美君/編譯
Aug 24 2000

Amiga和紅帽宣布合作發展Linux平台上的多媒體應用，這項合作計畫
，使紅帽能借助Amiga的SDK﹙System Developer Kit﹚，快速發展遊
戲、多媒體、Java-Based 等應用軟體程式。

------

開放原始碼的領導廠商紅帽公司，21日宣布與 Amiga的合作計畫，將
一起發展 Linux 平台上的高效能多媒體應用。 Amiga 所開發的 SDK
﹙System Developer Kit，系統發展工具﹚，有助於程式設計師快速
地研發各種遊戲、多媒體等應用軟體程式，並加強對Java的支援。

Amiga總裁暨執行長Bill McEwen表示，Amiga社群使用Amiga SDK致力
於遊戲和 Java 相關應用程式的發展，對於此領域有具體的表現，而
紅帽瞭解Amiga 所做的努力，也願意積極給予支援，是雙方合作的原
因。McEwen也指出，紅帽是Linux市場的領導者，也在Amiga SDK的發
展中幫助甚多，也是Amiga敲定這次合作機會的主因之一。

Amiga的發展工具SDK目前的最新版本是1.0版，Amiga SDK的基礎層，
使用虛擬處理器架構﹙Virtual Machine Architecture﹚，一支寫好
的可執行程式，可利用 SDK 移植到 Amiga 支援的其他系統平台上執
行， Amiga OS 可以在 Linux、Embedded Linux、Windows 95、98、
2000、NT、CE 和 QNX4 上執行，同時，Amiga-Based的應用程式，可
在 x86、PowerPC、M Core、ARM、StrongARM、 MIPS R3000、 R4000
、R5000、SH 3、SH4 和 NEC V850 處理器上執行。

新一代的Amiga SDK有獨特的Java 編譯機制及架構，所以 Java Byte 
Code執行的速度就像在本機執行的一樣快，Amiga SDK 零售價是99美
元，Amiga 也提供使用者 12 個月的免費更新服務及支援軟體。
