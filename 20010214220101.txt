■ TurboLinux與Linuxcare大幅裁員  為合併作準備

李孟諺/編譯
2001/02/12

在SuSE宣佈裁員後，9日傳出美商拓林思與Linuxcare也進行裁員與改
組消息。Linuxcare 裁撤大約十分之一的員工；美商拓林思則不願證
實多少員工遭到解僱。但此裁員消息，有可能是兩公司為了未來合併
而做的準備，Linuxcare已經公開表示期望與拓林思合併。

------

在SuSE宣佈裁減北美地區大部分員工後，美商拓林思與Linuxcare 同
時宣佈裁員與改組。美商拓林思行銷部資深副總裁 Jerry Greenberg
表示，因無法達到每季成長兩倍的預期目標，而做出裁員決定。至於
是否如外傳的裁減掉四十名員工，甚至有些部門整個被裁撤的消息，
GreenBerg不做任何回應。

Linuxcare方面，發言人Michelle  Nemschoff表示，該公司裁撤大約
十分之一的員工，大部分屬於行政管理或技術部門。 根據Linuxcare
執行長Arthur  Tyde最近表示該公司擁有一百五十名員工的數據來看
，大約有十五名員工下崗。

雖然兩家公司都公佈了裁員的消息，但另一方面兩家公司近日來卻為
了合併事宜進行密切商談。Linuxcare執行長Arthur Tyde表示，很期
望與Linux 軟體開發者拓林思合併。因為Tyde認為他的公司並不滿足
現在的成長率，而且服務為導向的公司應該與產品導向的公司合作。
為達到這個目標，他才會與提供教育、專業服務、與技術支援，且本
身又是Linux開發業者的美商拓林思洽談合併事宜。

Tyde也表示市場的發展並不如預期，他們許多的客戶是OEM 大廠，例
如 IBM、康柏或昇陽，很少有一般企業的客戶。因此，在整個市場方
向不如預期的情況下，他認為公司該做一些改變。

雖然目前為止兩家公司沒有達成最後的協議，但是Tyde有信心能妥善
處理好合併的細節。此外，合併之後，拓林思執行長Paul Thomas 將
會是合併後的新執行長？ 對此GreenBerg表示，營運一段時間之後，
現任營運長Ly-Huong Pham 將會接管整個公司。至於正確的合併日期
究竟是何時？拓林思訊息部副總裁 Lonn Johnston表示，每一天都是
合併完成日。

■ Caldera買下SCO的OpenServer產品線

沈美君/編譯
2001/02/12

最近資訊網路業似乎不太平靜，併購和裁員消息不斷。Caldera 上週
五宣布擴大對SCO產品線的併購計畫，除了UnixWare，也將買下SCO較
早版本的Unix平台OpenServer，擴展Caldera產品線。

------

最近Linux商業戰場動作頻頻，除了Turbolinux與Linuxcare的合併、
SuSE在北美分部大舉裁員， Caldera在2月9日宣布，將調整對Santa
 Cruz Operation公司﹙SCO﹚Unix產品線的併購計畫，原訂在 2001
年第一季可達成的協議，將延後至第二季。

Caldera公司在去年八月宣布買下SCO旗下的UnixWare，日前決定擴大
併購計畫，也買下SCO 較早版本的 Unix 平台「OpenServer」，總計
Caldera要付給SCO三千一百萬美金。

「改變併購案的主要原因，是省掉原來使用OpenServer的用戶在需要
技術支援時，不知道找哪家公司的麻煩。」Caldera 策略發展部副總
裁Benoy Tamang這樣表示。此外，Caldera 對未來軟體發展方向的主
控權也較大，可以鼓勵客戶轉到Linux平台，或者是UnixWare平台。

SCO從1999年停止發展OpenServer，今後Caldera 將會與SCO一起合作
，推出較為次要的更新套件﹙ updates ﹚。 在併購計畫改變之前，
Caldera只付了七百萬美元給SCO，計畫擴大之後，交易金額增加為兩
千三百萬美元，並且在往後一年﹙共四季﹚， Caldera每季都會付給
SCO八百萬美元，並且享有一千六百萬Caldera的股利。 SCO可以利用
這些資金，來發展即將由SCO改名為Tarentella的軟體事業。

Caldera計畫拓展產品線，從低階的Linux平台，到高階32-processor
多處理器的UnixWare平台，以及進階支援﹙例如叢集技術﹚，計畫在
不重新編譯的前提下，都可以在這些平台上執行Linux應用軟體。

■ Apache將整合Tcl語言

謝良奇/編譯
2001/02/13

以阿帕契網站伺服器著名的阿帕契軟體基金會， 近日宣佈阿帕契Tcl
計劃，將整合阿帕契網站伺服器及Tcl描述語言。 過去在網站環境上
廣受採用的Tcl ，未來合併世界最受歡迎網站伺服器阿帕契後，將可
成為該伺服器在後端描述作業上的最佳伙伴。

------

阿帕契軟體基金會﹙Apache Software Foundation﹚12日宣佈阿帕契
Tcl計劃，該計劃將以促進整合阿帕契﹙Apache﹚網站伺服器與Tcl描
述語言為目標，積極合併這一套擁有廣大使用群的開放原始碼描述語
言。

Tcl是一套跨平台的描述語言 ﹙Scripting Language﹚，自1988年由
Ousterhout 博士創造以來，在 網站、自動化測試等領域上受到廣泛
運用。易於學習、優秀的開發速率、優異的圖形界面支援、跨平台、
整合性強、彈性高等等，都是Tcl所以受龐大使用者社群喜愛的原因
。

在阿帕契Tcl計劃下，包括各項Tcl-Apache整合計劃，未來將合併阿
帕契伺服器的動力及Tcl描述語言具備的功能。Tcl-Apache組織將作
為這些專案的參考資源匯集。阿帕契Tcl首先展開的mod_dtcl專案，
可供阿帕契伺服器使用，是一套快速而輕巧的Tcl實作。

mod_dtcl專案是伺服端解譯式Tcl的開放原始碼實作，在阿帕契下執
行，可允許緊密整合HTML和Tcl。 mod_dtcl透過額外的Tcl模組，載
入後更可達到建立影像、存取資料庫等其它功能。

阿帕契Tcl副總裁及mod_dtcl發起人David Welton表示，Tcl 輕巧、
易於學習，且在使用、開發及測試上皆行之有年，對於網站而言Tcl
是相當重要的語言，此外，Tcl亦擁有廣大使用者基礎，是相當活躍
的開放原始碼計劃。

■ Linux的新大陸─路由器市場

謝良奇/編譯
2001/02/12

繼伺服器市場後，Linux 下一個要進攻的對象是什麼？答案是路由器
。網路處理器技術登場後，路由器龍頭思科公司，以往擁有的競爭優
勢已慢慢消失，在網路設備市場擴張後，結合 Linux和標準硬體的開
放路由器正悄悄上路。

------

網際網路急速成長，造成網路市場空前熱絡，因此對各類型網路設備
行程大量需求，其中可稱為網際網路心臟的重要設備：路由器，正成
為Linux大顯身手的下一個目標。

過去路由器市場幾乎由擁有80％市佔率的思科﹙ Cisco﹚獨霸，如今
在路由器市場持續擴張下，已出現許多競爭廠商。網路設備的新開發
模式是這些廠商重要的競爭籌碼，網路處理器是其中最關鍵的技術。

新穎的網路處理器技術具有許多設計上的優點，可取代思科以往開發
產品時採用的矽定製技術，包括英特爾、摩托羅拉在內的多家廠商皆
已跨足網路處理器市場。網路處理器以類似標準處理器的設計，允許
廠商即使面對快速演進的標準，仍可迅速為系統添加功能，比起昂貴
更耗費多餘設計時程的定製晶片，要優越許多。網路處理器降低路由
器進入市場的障礙，一度是網路路由器龍頭的思科，其領先優勢正逐
步被削弱。

為取得更佳的競爭能力，新興的路由器廠商需要標準軟體來搭配標準
硬體，儘管Linux 無法直接在網路處理器上執行，但路由器內包含的
控制處理器對 Linux而言是相當恰當的位置，同時也是一個絕佳的機
會。Linux 以完善工具集、原始碼可用性等優勢，將可在控制處理器
軟體中扮演吃重角色。

在控制處理器的應用上，多數廠商傾向自行開發軟體，但近來有不少
廠商採用Linux的開放環境，像是MMC日前與MontaVista合作，展示一
套完全採用標準晶片及 Hard Hat Linux的開放路由器平台。用Linux
，OEM 廠商可迅速開發出能與思科產品競爭的網路系統，相對於標準
硬體可加速產品製造時程，Linux 在廠商贏取競爭的產品功能開發方
面，佔有關鍵地位。

網路市場擴張連帶為 Linux製造更多機會，即使在此市場中還有另外
一些對手，如傳統即時作業系統廠商等等，但鑒於控制處理器軟體開
發動輒數百萬行的程式碼，Linux 在成本、開發時程上的優勢，是不
可或缺的要素。

■ 微軟將開放部分視窗原始碼  延攬Linux人才

李孟諺/編譯
2001/02/13

微軟公司12日表示他們將開放視窗的原始碼給部分公司使用。此外，
微軟也在澳洲公開徵求Linux 程式高手，準備發展下一代視窗系統。
這兩個行動是否代表微軟的策略將轉向Linux ，並開始接受開放原始
碼？或是只是為了維持微軟在作業系統市場的優勢？

------

微軟對於開放原始碼一直以來持保留的態度，但是在12日一位微軟官
方人士指出，微軟公司將宣佈有限度的視窗原始碼分享計劃，而且細
節將在這幾個星期內宣佈。此外，微軟也已經在澳洲開始尋找 Linux
高手，幫助他們發展下一代的視窗作業系統。

但是，微軟的伺服器軟體產品經理Doug Miller 強調，開放部分軟體
原始碼並不代表微軟想要進入開放原始碼軟體的領域。而且，跟許多
軟體廠商一樣，微軟提供原始碼給廠商或是使用者已經有多年的時間
。只是直到現在，這個措施一直低調進行，而且屬於機密協議。上週
，在拉斯維加斯舉辦的視窗嵌入式系統開發者會議，微軟宣稱一些公
司將能夠取得視窗CE的原始碼。使那些公司可以讓視窗CE更符合他們
的需求。

Miller也表示，藉著提供原始碼，資訊家電維修者可以從原始碼內找
出解決問題的管道。一些視窗使用者如果能夠使用原始碼，那他們可
以從根本上去找出問題的來源。

除了開放原始碼給部份業者，微軟對頭號敵人的Linux 也開始有部分
動作出現。由微軟總部直接發出的人才招募消息，鎖定對Linux 有研
究的專家。而且微軟已經與雪梨、莫爾本、阿德萊德的一些Linux 團
體接觸，尋找對加入微軟團隊有興趣的軟體開發者。他們尋求的對象
條件是碩士畢業，主修電腦科技或相關領域，而且熟稔C語言與C++。
微軟也透過Email 邀請開發者參加下個月在雪梨的面試。

澳洲軟體發展協會總經理表示，微軟承認他們需要一些新的技術來補
足他們不足的地方。而且從他們的角度來看，如果那些批評視窗的人
能夠加入團隊，對於現在的團隊來說是會有一個很好的平衡效果。我
們也非常樂於見到微軟承認Linux的優越性能。

■ OpenMPEG計劃─發展GPL授權的MPEG4軟體

張書銘/編譯
2001/02/12

數位多媒體影音應用是目前資訊科技上應用甚廣的一門技術，目前在
多媒體技術上最新的 MPEG4標準提供了較過去標準更好的壓縮比與取
樣品質。由於開放原始碼界尚未推出可支援此標準的軟體，在開放原
始碼社群人士的號召下，目前正推行一項OpenMPEG計劃，讓最新的技
術能應用在開放原始碼的軟體上，讓自由軟體使用者也能享受到最新
技術的便利。

------

目前在多媒體技術上最新的 MPEG4標準提供了較過去標準更好的壓縮
比與取樣品質，但由於開放原始碼界尚未推出可支援此標準的軟體，
因此開放原始碼界的人士便開始著手推動OpenMPEG計劃，開發一套符
合GPL授權且支援MPEG4標準的自由軟體。

MPEG4技術﹙第4代動態影像解壓縮協定﹚在目前提供了最佳的畫質與
檔案壓縮比，以一張標準容量的CD-ROM光碟，就可存放約八十分鐘相
當於DVD﹙使用MPEG2標準﹚畫質的高清晰度電影節目。因此即使沒有
DVD播放器，只要使用光碟機也可以享受到高品質的影音效果。

目前較為知名可支援MPEG4標準的軟體，有 Windows Media Player﹙
需安裝MPEG4代碼─DivX Codec plugin﹚與PowerDVD等。但都不是開
放原始碼的自由軟體，因此開放原始碼社群人士便發起OpenMPEG開發
計劃，希望推出一套符合 GPL授權的軟體，讓使用者有選擇的自由。

OpenMPEG組織表示希望更多熟悉數位影像處理新技術的開發者能加入
此專案，也指出OpenMPEG最大的問題在於 MPEG4標準中有部分技術屬
於他人的專利，因此取得握有專利權人士的支持，將是這項計劃成敗
的重要關鍵。

   http://www.advogato.org/article/241.html
2、關於OpenMPEG
   http://www.openmpeg.org/about/

■ Linux影音新軟體─採GPL的popCorn

梁天佐/編譯
2001/02/13

不論是對進階使用者，還是普通的電腦玩家，影音播放軟體早就成為
作業系統的基礎配備；隨著Linux 的普及，也有愈來愈多影音應用軟
體支援、或專為Linux量身打造。 popCorn的加入讓Linux使用者又多
了一項不同的聲光選擇。

------

popCorn是一家位於法國南部的軟體公司eProcess專為Linux所設計的
影音播放軟體，在視窗桌面環境下使用，操作介面模仿一般視聽機器
，功能齊全，除播放/暫停/停止、音量大小這些基本操作之外，也提
供音效調整、直接存取電影與播放目錄管理等功能，與一線影音播放
軟體具有相同的水準。

軟體本身主要的設計是用來播放MPEG-1 規格影像、與MP3音效的檔案
格式，為能存取網路影音檔案，並且相容RTSP協定，可直接播放來自
網路上媒體伺服器所提供的串流資料。

popCorn以GPL方式發行，使用者可自行下載包括原始碼在內所有的相
關檔案，程式碼以C語言撰寫，雖然目前發展狀態仍處於 0.1 Beta版
，但已可正常使用；eProcess正於網路邀集各界意見與測試除錯中，
並且就 popCorn計劃的移植、改寫、研發技術提供支援與訓練等加值
型態的服務。

eProcess公司的專業領域在於研發如 VOD客戶端多媒體技術、媒體廣
播、互動式電視與Content-on-Demand 等解決方案，特別是這些項目
在嵌入式設備上的應用；該公司的市場目標鎖定於提供嵌入式設備如
互動電視、訊號解碼器、數位 VCR、機上盒與其他會應用多媒體器具
的製造商解決方案。

   http://linuxpr.com/releases/3317.html

■ Interbind釋出XML Messaging Server0.8版

沈美君/編譯
2001/02/13

橫跨異質協定、資料格式的服務，在充斥形形色色運算平台的網路上
，的確是迫切的需要。Interbind宣布釋出「XML Messaging Server
」，可處理異質協定、不同資料格式的傳送與轉換，版本0.8 已開放
免費下載。

------

Interbind公司於2月12日宣布釋出「XML Messaging  Server」第0.8
版，是專為以Web為基礎的服務而設計的傳訊平台，以點對點﹙peer-
to-peer﹚及XML為基礎架構，可自發展者網站免費下載。 

以Web服務導向及點對點通訊為架構設計的 「XML Messaging Server
」，適用於任何支援Java的運算裝置，「XML Messaging Server」遵
循W3C的XML通訊協定，可處理異質網路、異質通訊協定﹙例如Secure
 Sockets、HTTP、SMTP 等﹚間的訊息，因此不但可擔任同質系統間
的直接訊息傳送，或者異質系統間的轉送，更能依據schema﹙資料綱
要格式﹚，轉換不同的訊息格式。

「XML Messaging Server」支援多種訊息資料來源，包括關聯式資料
庫、XML、各種檔案系統、試算表、網頁服務資料、甚至一般的 Java
 API。除了易操作的Web使用介面，管理本機或遠端「 XML Messagin
g Server」伺服器，在安全性考量上，「XML Messaging Server」相
容於一般業界安全機制的實作，例如：公開金鑰加密技術﹙public-k
eycryptography﹚、秘密鑰匙交換﹙secret key exchange﹚， 也支
援secure sockets及security policy，來提高系統安全性。

要安裝「XML Messaging Server」，必須注意安裝平台是否支援SAX2
或DOM物件的XML parser﹙XML解析器﹚，也必須具備昇陽的Java標準
執行環境﹙Runtime Environment﹚，發展「XML Messaging Server
」相關應用，JDK1.2或以上的版本是必備的工具。

執行效能佳，且易於建置、調校的「XML Messaging Server」，可由
發展者社群網站下載原始碼，此外，網站也有關於此伺服器的安裝文
件、教學、範例程式，讓使用者快速學習「XML Messaging Server」
開發應用程式。發展者網站也有專為「XML Messaging Server」設置
的討論區，Interbind相當企盼來自XML社群的使用心得分享。

相關網址：
1、Interbind釋出「XML Messaging Server」0.8版，可免費下載
   http://www.newsforge.com/article.pl?sid=01/02/12/2116243&mode=thread
2、關於Interbind的XML Messaging Server
   http://www.interbind.com/xml_server.jsp
3、發展者社群網站
   http://www.developer.interbind.com
