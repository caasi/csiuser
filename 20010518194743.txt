■ 最新資料庫效能測試  Linux大幅超前Windows 2000

梁天佐/編譯
2001/05/17

開放原始碼軟體中，發展起步較慢的資料庫領域最近傳出捷報，據提
供業界資料庫解決方案測試參考數據的組織Transaction Processing
Performance Council﹙TPC﹚最新公佈的測試資料，SGI執行Linux作
業系統、搭配IBM資料庫軟體DB2  7.2系統，大幅度領先執行Windows
2000、搭配微軟SQL Server 2000的系統。

------

TPC 最新的資料庫軟體應用測試數據，又一次提供了反駁微軟攻訐開
放原始碼作業系統不適合商業應用的證據。據最新100G等級資料庫效
能測試數據結果，SGI執行Linux並搭配IBM DB2  7.2的組合，與其它
九部不同硬體，但均執行Windows  2000、搭配微軟SQL Server 2000 
的系統相較排名第一， TCP-H測試評分是2733，大幅領先第二名康柏
Proliant 8000-X700-8P系統的1699。

TCP是以TCP-H測試標準進行這項測試， TCP-H是一種檢驗在嚴苛的商
業要求環境下，資料庫軟體執行高複雜度資料查詢，與大量驗證資料
能力的標準， TCP-H所提供的數據是以效能為標準，反映系統進行資
料查詢的各種能力。 TPC定期公佈的測試數據，常是企業採用資料庫
系統的重要參考。

IBM對TCP的測試結果感到滿意，表示DB2資料庫系統原來只在IBM的硬
體上使用，但因近來支援Linux 的政策，調整之後的DB2也能在Linux
上執行，最新的DB2 7.2就加入了如向量讀取﹙Vectored-Read﹚機制
之類，讓Linux 更具彈性的功能。 IBM效能與進階技術工程師兼經理
Bermi  Schiefer說：「我們對協助將Linux帶入主流有興趣，而之前
TPC一直沒有Linux的測試數據。」。

另一方面，微軟強作鎮定，微軟SQL Server團隊的產品領導經理Jeff
Ressler表示，在 100G等級領域裡微軟已經稱霸太久，終於有競爭者
是一件有趣的事。Ressler雖然接受TCP的測試結果，卻仍強調測試的
硬體基準不甚公平，SGI的硬體是四部機器的叢集，而執行Windows20
00的只有一部單機，微軟還會尋求其他測試方式。

資料庫確實是開放原始碼軟體較弱的一環，雖然MySQL與 PostgreSQL
還算完整，但還在起步階段，也缺乏實際應用的經驗，因此商業級資
料庫應用還需要IBM之類有經驗的公司提攜。

TCP是以SGI 1450伺服器執行核心為2.4.3的Linux版本進行測試，DB2
7.2將在6月8日上市，SGI-Linux-IBM組合的系統則可望在10月31日後
商品化，正式在市場上推出。

■ Eazel資金募集不順宣佈倒閉

李孟諺/編譯
2001/05/16

去年甫成立於美國加州的Eazel公司，在3月間傳出裁員消息之後，於
15日宣佈結束營業。這家由麥金塔好手與一些資深電腦人共同創立的
公司主要從事開發GNOME 桌面環境套件，但由於資金募集不順，面臨
倒閉的命運。

------

由美國線上與蘋果公司等電腦業界知名專家所共同創立的Eazel 公司
，在11日就傳出即將倒閉的消息。雖然最近幾天仍傳出公司高層仍努
力與昇陽等公司和一些投資人交涉，希望能募集資金。但是仍然回天
乏術，終於在15日宣佈關閉。

Eazel公司創辦人包括麥金塔原始設計者之一的Andy Hertzeld、曾募
集三十億美元，創立蘋果電腦軟體推廣集團的Michael Boich 、曾任
麥金塔軟體經理，也曾身為Java程式語言設計者之一的 Bud Tribble
，以及美國線上前資深副總Mike Homer，陣容可說十分堅強。

Eazel成立的宗旨是希望打造一個使用者友善的Linux的圖形使用者界
面，讓Linux系統可媲美如麥金塔或微軟視窗的作業系統。並以GNOME
桌面環境為主要開發對象。該公司雖然成立之初募集資金一千五百萬
美元的資金，但是卻遭受外界質疑資金運用失當。在發表鸚鵡螺檔案
管理程式﹙Nautilus﹚之後，立刻宣佈裁員四十人，讓外界對該公司
的財務狀況產生懷疑。

早在11日就已經傳出Eazel 即將倒閉的流言，期間也傳出昇陽撤資的
消息。不過Eazel執行長Mike Boich表示，Eazel從未公開宣佈過擁有
來自昇陽的資金，而昇陽也對此說法做出回應。但是Eazel在 14號的
理事會招開之後，隔天便對外宣佈倒閉的消息。Eazel 創辦人之一的
Bart Decrem表示，在最近幾個月當中，Eazel已經嘗試募集資金，但
是終究挽回不了公司面臨倒閉的命運。

Eazel 這家公司的壽命僅十六個月，而且鸚鵡螺﹙Nautilus﹚檔案管
理程式也變成了唯一的代表作。而至於倒閉的原因，有人批評 Eazel 
資金管理不當，也有人認為成立該公司的成員都吝於拿出自己的資產
投資。但不論原因為何，GNOME 使用者社群與其發展都受到該公司倒
閉的影響，而Eazel 目前能做的，只能延緩網站關閉時間，並請使用
者盡速取回在Eazel線上儲存服務當中的資料。

■ Rebel資金吃緊，裁員23％

沈美君/編譯
2001/05/14

才剛在2月下旬推出新款網路伺服器的Rebel公司，12日宣布裁撤23％
的員工，主要因為獲利不足以平衡其他部分損失，且資金募集困難。
但執行長John Kelly強調，公司不會宣布破產，近期內也不會再裁員
，並預期在接下來兩個月內，將有一位新的投資者加入。

------

景氣未見好轉，資金籌措對資訊業者而言，是難上加難。加拿大渥太
華的伺服器廠商Rebel 公司因為資金募集困難，加上特定部門的銷售
策略未達到損益平衡，於12日宣布裁撤23％的員工。

Rebel裁撤近1/4 的員工，大部分都是NetWinder伺服器部門的行銷人
員。「NetWinder伺服器部門」是由「Hardware Computing Canada﹙
加拿大硬體運算﹚」改名而來，在1999年從Corel手中被Rebel購買，
目前Corel仍持有Rebel.com25％的股份。

今年2月下旬，Rebel發表採全美達Crusoe 400 533MHz 處理器的網路
伺服器「NetWinder 3100」，提供完整網路工具，可處理電子郵件系
統、網路管理、防火牆等工作，小而美的性質對中小型企業而言，可
以降低購買不同伺服器系統的成本。

雖然NetWinder 伺服器部門的服務業務呈現持續成長，公司也正積極
將此部門的銷售策略改為直接銷售，但因為NetWinder 伺服器部門的
獲利不足以平衡其他部分的損失，促成這次裁員的重要原因。為穩定
其他投資人及員工的信心，Rebel執行長John  Kelly強調，公司不會
宣布破產，近期內也不會再裁員，並預期在接下來兩個月內，將有一
位新的投資者加入。據消息指出，Rebel所期待的新投資人可能是IBM
，但Rebel公司發言人不願證實此消息。

除了裁員動作， Rebel將重整公司架構為兩個分部，原來負責企業服
務的部門，也就是Rebel 賴以獲利的核心事業，將合併成為其一；另
外的電腦硬體部門，也就是裁員的主要部門，將不會出售，而是將銷
售通路模型加以重新定位。

■ 開放原始碼與資本獨占，孰輕孰重？

梁天佐/編譯
2001/05/16

微軟資深副總裁Craig Mundie在紐約大學發表攻擊開放原始碼自由軟
體的演說之後，開放原始碼界的重量級領導人物除各自演說與撰文回
應外，也覺得以往各自為政的社群在這件事上，有必要採取一致對外
口徑，於是由社群重量級人物Bruce Perens代表發起的行動，得到了
Richard Stallman、Eric Raymond與托瓦茲等人物的連署。

------

微軟砰擊開放原始碼軟體，或許是因為感受到自由軟體未來對微軟的
威脅，然而開放原始碼社群或個人各自的回應有時難免不夠周全。有
鑑於此，開放原始碼社群領袖簽署了由社群重量級人物Bruce Perens
發起的團結行動，社群將對微軟採行一致的回應，以免遭各個擊破。

Bruce Perens發起文中透露，行動的主要原因是：由Craig Mundie的
演說來看，微軟顯然已經正式將開放原始碼自由軟體當成商業上的敵
人，但是開放原始碼卻沒有托辣斯企業縝密的侵略手腕與壟斷個性，
目的不在毀滅對手，只是期望個人與企業如IBM 、惠普等公司一起公
平競爭，因此對商業軟體公司的攻擊，有一致對外的必要。

在Craig Mundie諸多攻訐中，首先要正本清源的就是微軟刻意誤導自
由軟體﹙Free  Software﹚的定義，「Free」在此是指自由而非免費
，與微軟所謂.com公司免費提供產品與服務，以致失敗是完全不同的
事。在GPL的授權定義下，任何人若採用GPL發行的原始碼，就表示在
開發自己的軟體時，已經省卻了從零開始的費用，開放原始碼社群以
此交換新軟體相同的開放要求，非常公平。

其次在Craig  Mundie指控GPL妨害軟體產業發展的部分，以開放原始
碼在企業最成功的應用實例─阿帕契伺服器來說，就算是沒有採用開
放原始碼的企業也在受惠。另外，GPL要求的是以GPL再發展出來的軟
體必須以GPL方式發行，並非微軟所謂，一旦公司採行GPL就必須將所
有的軟體以GPL方式公開；事實上以GPL 方式發展的軟體如GUI桌面﹙
Gnome與KDE﹚，發展不到四年，已經被工作站廠商如昇陽、惠普正式
取代CDE ，足見開放原始碼能夠有效且快速地發展軟體，促進軟體產
業發展。

至於商業經營開放原始碼軟體的公司，它們可以藉由社群的協力降低
軟體開發成本、提昇服務效率、提供訓練諮詢等方式經營，或許目前
金額不如扥拉斯聚斂得那般龐大，但至少相當公平，使用者不必被軟
體供應商箝制。這也不乏如紅帽、 SuSE等成功實例，大型公司如IBM
、戴爾也積極朝開放原始碼發展中。

在微軟高唱所謂安全性與相容的問題上，答案更是明顯。目前沒有一
個系統如微軟的產品那麼容易受到病毒攻擊，有那麼多的安全性漏洞
與後門可鑽；開放原始碼軟體或許一開始也有問題，但在廣大社群與
使用者的驗證之下，幾乎都能迅速修正。在所謂的不相容的問題上，
微軟自己就是不相容的最大製造者，為了強迫使用者升級，新版Word
的格式硬是不能相容舊版就是最好的例子。

簽署Bruce Perens倡議文的重量級人物與社群有：Richard Stallman
、Eric Raymond、托瓦茲、Miguel de Icaza﹙Gnome GUI桌面計劃﹚
、Larry Wall﹙Perl創作者﹚、Guido van Rossum﹙Python創作者﹚
、Tim O'Reilly﹙O'Reilly出版﹚、Bob  Young﹙紅帽創辦人﹚以及
Larry Augustin﹙VA Linux執行長﹚。

■ Mac OS X將推出伺服器版本

翁千婷/編譯
2001/05/16

蘋果推出新版伺服器用的Mac OS X作業系統，將於21日的蘋果國際研
討會上發表正式版本。此伺服器主要提供系統管理、網站管理與網路
服務的特色。

------

除了將Mac OS X應用在個人電腦上，新版蘋果伺服器也打算採用 Mac 
OS X作業系統，現行的作業系統版本將是最後的測試版本。如無意外
，正式版本將會在5月21日的蘋果國際研討會上釋出。

這款伺服器的重點放在網路管理，提供Mac 、Windows、Unix跟Linux
客戶端的服務、PostScript檔案列印、網際網路與網站服務，較以往
的伺服器有幾項差異：最大的差別，在於第一次使用水系圖形介面﹙
Aqua graphical interface ﹚，對過去的Mac使用者來講是一個全新
的使用介面。

另一個特色是圖形介面的功能管理與設定。電子郵件、 QuickTime串
流影像伺服器與DNS 的設定與修改，都可透過使用者介面來完成，不
需像過去編輯文字設定檔。網站服務方面，增加支援JAVA  Servlets
與PHP、 IP過濾跟網域名稱伺服器等項目的新功能。此外蘋果計畫修
改Mac 的管理功能，提供網路管理服務，可以透過網路設定使用者對
應用程式、根目錄與印表機的存取權限。

之前備受爭議的蘋果公開原始碼授權Apple Public Source  License
﹙APSL﹚也於近日獲得Open Source  Initiative﹙OSI﹚投票通過認
同為可接受的開放原始碼授權。一場開放原始碼授權之爭，可望就此
告一段落。

■ SuSE與intraDAT結盟  提供VShop電子商務解決方案

李孟諺/編譯
2001/05/17

德商SuSE公司與開發Linux 平台電子商務軟體的intraDAT國際於16日
對外宣佈，透過SuSE的事業夥伴計劃建立合作關係。該計劃主要在吸
引SuSE的客戶加入已有的加值轉售者與整合者的行列，並推廣該公司
的主力產品VShop。

------

SuSE公司與intraDAT國際16日對外宣佈，透過SuSE的事業夥伴計劃﹙
Business Partner Program﹚建立合作關係。身為SuSE的最新夥伴，
IntraDAT將參與SuSE全球共同行銷與產品支援計劃，並推廣該公司的
主力產品VShop。

IntraDAT國際是與SuSE同為誕生於德國的IntraDAT於美國成立的子公
司，是發展Linux 平台電子商務網站領域中的領導者。目前該公司的
主要產品是以自行研發的VSL編譯語言為基礎開發的VShop，是目前最
快而且最穩定的電子商務中介軟體﹙middleware﹚。

SuSE總裁Dirk Hohndel表示，目前全球有超過一千五百家網路虛擬商
店使用內附於SuSE Linux的 VShop。此軟體主要著重在整合與維持中
小型電子商務的B2B平台，並能支援七種不同的語言。 intraDAT國際
的執行長Ralf Schwoebel也指出，多年來SuSE已經透過加值服務轉售
者與整合關係結合歐洲頂尖的企業。透過與SuSE的合作關係，該公司
能夠提供給客戶更好的服務與完整的解決方案。

而促成此樁合作案的事業夥伴計劃，主要為了鼓勵SuSE的客戶加入該
公司已有的加值轉售業者﹙VARs﹚與整合的行列。並吸引新的加值轉
售者與整合者的加入。

SuSE的事業夥伴計劃根據不同加值轉售者與整合者有三種等級。一是
轉售者方案，此等級提供轉售者與系統管理者免費的延伸產品資訊與
行銷與材料的支援。其次是 SuSE 加值夥伴方案，加值轉售者使用的
Linux解決方案可以接受額外的Linux支援，包括成立訓練課程與工作
坊，安裝支援、即時通訊和夥伴網站的服務。再者，屬於SuSE優良事
業夥伴等級的客戶，可以額外的使用進階版本的SuSE Linux產品，並
透過SuSE計劃支援與共同行銷活動推廣產品。

■ 電腦技術文件計畫「CTDP」起跑！

沈美君/編譯
2001/05/16

基於提倡技術文件自由流通、自由取得的精神，「電腦技術文件計畫
」﹙Computer Technology Documentation Project,CTDP﹚於15日宣
佈正式起跑，CTDP網站以簡單的目錄設計，提供適合初學者及進階使
用者的資訊技術文件，使用者手邊若有值得參考的資訊，亦可自由上
傳提供給CTDP計畫。

------

CTDP的成立初衷，不約而同與Linux文件計畫﹙Linux Documentation
Project ﹚的精神相仿，CTDP計畫的首要目的，就是倡導適合初學者
及專家使用者的技術資訊、文件之自由交換，「藉由概念與文件的分
享交換，將能提升個人甚至於團體的生產力。」這也是CTDP的核心信
念所在。

此外，CTDP的成立是為了提供資訊技術文件給沒有能力負擔費用的使
用者。CTDP為非營利組織，CTDP計畫所提供的技術文件大抵也是以不
收費為基礎，但組織也提供了另外一種選擇，讓提供技術文件的作者
也能經由撰寫文件來獲取應得的報酬。不過，CTDP仍是處於不主動提
倡商業仲介的立場。

CTDP計畫所定義的技術文件和資訊，不分「黨派」與作業系統，只要
是與電腦技術相關的文件資訊，都在CTDP定義的範圍內。CTDP計畫所
涵蓋的技術文件及資訊，包括網頁標記語言、作業系統、硬體、程式
語言、網路等領域，從初學者到進階級的專家都適合閱讀，這些文件
和資訊，將以評論、專欄、秘訣、相關網站位址的形式來表現，深度
與廣度兼備，目的要讓取用資訊的使用者，在最短的時間內作有效率
的學習。

目前在CTDP網站上可取得的技術文件，包括關於Windows NT、Linux
及幾份關於網路的文件，和關於XML、HTML、CGI、SSI、PHP與電腦硬
體等的簡易指引。若使用者手邊有值得參考的技術文件，也歡迎提供
給CTDP計畫作參考。
