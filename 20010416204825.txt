■ 舊金山灣區捷運系統採用Linux解決方案

沈美君/編譯
2001/04/13

舊金山灣區捷運系統﹙BART﹚於兩年前開始升級其電氣化呈示系統，
成功以Motif作為開發工具、並以Linux取代原來使用的昇陽工作站，
在4月份的Linux Journal中將更詳細介紹BART的開發決策與經驗。

------

Integrated Computer Solutions 公司﹙簡稱ICS﹚於4月12日宣布，
舊金山灣區捷運系統﹙San Francisco Bay Area Rapid Transit，簡
稱BART﹚改版電氣化呈示系統﹙Electrification Display System﹚
，並且選用Linux，來替代原來的昇陽Solaris系統平台。

BART的電氣化呈示系統，用以追蹤由Pacific Gas and Electric而來
的高壓DC電力，如果遇到緊急狀況，必須撤離電車，這個系統可用來
切斷電力，確保乘客安全離開。此電氣化呈示系統，原來包含連接於
Tandem S4000的數部 NCD X終端機，以及控制中心內的一兩部昇陽工
作站。

1998年底，因應實際需要，BART的電氣化呈示系統必須作系統升級，
因為選用新力的大螢幕，原來採用的昇陽工作站無法支援，經過評估
，發現Linux的 X Window只要經過簡單調校，就可以支援新款螢幕。

另外，在應用程式開發方面，BART 選擇Open Motif作為GUI﹙圖形使
用介面﹚開發工具，採 ICS公司旗下研發的「Builder Xcessory PRO
」來撰寫應用程式使用介面，可針對使用者的輸入，快速建置相對應
的輸出畫面，並採用LOOX Software 公司旗下的「LOOX++」，發展應
用程式的視覺化面向，也更容易建置系統模型的圖形表示。BART之所
以棄Qt或GTK+，反倒選用Open Motif，是因為原來在UNIX上的應用程
式，即以Motif開發，BART的工程師對Motif也比較熟悉，對於緊迫的
時程而言，以Motif為第一選擇是再適合不過。

系統完成後，經過事後評估，BART重新建置電氣化呈示系統，節省了
約15-20％ 的預算，因為不用購買作業系統的使用授權、昂貴的工作
站、使用快速的開發工具。

■ 巴西利用開放源碼打造資訊資訊生活化環境

張書銘/編譯
2001/04/13

資訊化已成為鑑別國家發產程度的指標之一，因此許多國家陸續進行
資訊產業與資訊生活化的發展計劃。但由於國情與經濟狀況的不同，
成本問題成為首要的考量因素。為了降低資訊成本，使其深入一般民
眾家庭，開放原始碼資源成為許多國家的首選。

------

目前世界各國不斷推動資訊深入民間的計劃，以期提高國家競爭力；
為了降低成本，讓更多民眾能取得這類資源，因此開放原始碼的精神
與產品受到極大的期待。 巴西政府日前就與國家半導體﹙ National
 Semiconductor﹚公司以及Samurai公司進行合作協商，希望推動低
成本的資訊家電計劃，讓網路能深入一般巴西家庭。

根據巴西政府的統計，目前巴西一億七千萬人口中，大約只有三百九
十萬人能夠使用網際網路，而且集中在大型都會區內部，資訊化程度
薄弱，因此巴西政府急於尋求快速便宜的解決之道。經過巴西 Minas
 Gerais州立大學的評估後，巴西政府選擇知名資訊家電供應商國家
半導體、及曾負責巴西電子投票系統的 Samurai公司為合作對象，共
同開發解決方案。

在這項解決方案中，主要將以兩家公司共同開發的 Samurai 2000 IA
系統為基礎。Samurai 2000除了可應用於一般 thin-client機器，也
可以配合內建數據機的資訊家電產品，只要使用 Samurai FlashCard
進行身分認證後便可自由存取網路資源。同時由於它以 Linux為作業
系統，因此在系統與應用程式成本上都可以更為低廉，對民眾更具吸
引力。

目前這項計劃已經正式展開，預備推動巴西全國的資訊化解決方案，
國家半導體與Samurai公司兩家公司，積極地與民間ISP業者以及通信
公司聯繫，以取得必要的技術與網路設備支援，國家半導體公司南美
區理事長表示，只要能獲得ISP與通信業者支援， Samurai 2000有信
心以最低廉的價格，讓鄉村區的巴西民眾享受到網路資源。

■ SuSE Linux 7.1推出Sun Sparc版本

張書銘/編譯
2001/04/13

SuSE公司正式推出SuSE Linux 7.1 for Sun Sparc處理器版本，除了
延續在其他版本上的特性，這個針對昇陽處理器所設計的版本更加強
網路應用效能，並提供更親切的使用介面。因此無論一般使用者或是
網管人員，都可藉由使用SuSE Linux 7.1 for Sun Sparc處理器版本
快速建置屬於自己的Linux使用環境。

------

SuSE Linux 7.1的Sparc版本正式出爐。針對Sparc處理器推出的版本
中，SuSE特別針對網路效能以及使用者介面做了改進，讓 Linux在使
用上更為簡單。而挾著之前推出針對 x86處理器架構版本的狂銷，公
司表示Sparc版的推出，將更有助 SuSE Linux成為全球最值得信賴的
開放原始碼軟體。

SuSE 7.1提供了多種供一般使用者或網管人員使用的工具，包括 KDE
2.0.1、Linux 2.4或 2.2.18核心、glibc 2.2.0、Netscape 6、CUPS
列印系統等，同時可選用 XFree86 4.02或3.3.6版等套件。此外SuSE
7.1也開始提供LFS﹙Large File Support﹚以及IPv6的支援，因此無
論架設伺服器或個人使用都很適合。

而SuSE Linux 7.1 for Sun Sparc處理器版本中，特別強調易於使用
的特性，因此特別加入新的 YaST2安裝與系統管理工具組。它提供了
安裝來源切換﹙可選擇光碟或網路安裝﹚、磁碟切割、Dial-out防火
牆功能、Network routing editor以及多種列印裝置組態控管功能。
將常用的網路以及周邊設定全部整合至單一管理工具內，讓使用者能
夠在統一的介面下進行各種管理與應用。

此外，SuSE Linux 7.1 for Sun Sparc處理器版本使用的 KDE 2.0.1
中也加入了新的檔案管理系統 Konqueror與KOffice，Konqueror整合
了瀏覽器以及檔案總管的功能，簡化使用者的操作環境；而 KOffice
內則提供了包括文字編輯器KWord、試算表軟體KSpread以及影像處理
軟體KPresenter，讓一般使用者擁有應付日常工作必須的應用軟體。

■ MontaVista釋出熱抽換技術規格原始碼

沈美君/編譯
2001/04/12

不單是軟體要開放原始碼，現在硬體產業也盛行將技術規格開放，透
過協同發展的機制，達到加速技術標準普及的目的，MontaVista公司
宣布將「 PICMG 2.12 軟體介面規格」釋予開放原碼社群，欲使高可
用性的關鍵技術，儘速在市場上被普遍採用。

------

MontaVista公司於10日宣布將熱抽換﹙Hot Swap﹚技術規格原始碼，
經由SourceForge 網站貢獻給開放原始碼社群，目的除了加速產業界
接受此規格之外，更欲提倡發展高可用性與當機備援的架構，讓此關
鍵技術儘速在市場上被普遍採用。

這項技術被稱為「 PICMG﹙PCI Industrial Manufacturers Group﹚
 2.12 軟體介面規格 」，定義了如何在電腦仍持續運作的狀態下，
將CompactPCI從系統抽出或安裝進去的過程，且不需要關閉電源和重
新啟動電腦。這份規格包含熱抽換的概念架構、也定義軟體如何透過
介面與作業系統溝通，達到支援熱抽換的目的。簡而言之，PICMG 2.
12的架構支援對熱抽換機制的設定、調校與管理功能。

MontaVista工程部副總裁Kevin Morgan 指出，開放PICMG技術規格，
目的在邀請開放原始碼社群，與眾多硬體領導廠商，一起將 PICMG標
準實作到各種可能的平台應用面上。MontaVista公司所公布的規格，
基於Linux核心2.4規格，也包含Motorola Computer Group 82xx高可
用系統、Ziatech ZT5550的平台驅動程式。至於「PICMG 2.12軟體介
面規格」，可由SourceForge網站下載。

在開放原始碼嵌入式系統市場，持續發展軟體解決方案的MontaVista
公司，創立於1999 年，致力於提供Linux作業平台，以及嵌入式系統
的發展工具，旗下所發展的「Hard Hat Linux」，是專為開發嵌入式
解決方案的作業平台。

■ CORBA新用法，解決開放原始碼授權限制

翁千婷/編譯
2001/04/02

需要公開原始程式碼的開放原始碼授權，讓程式開發者對於使用開放
原始碼軟體做為解決方案一事望而卻步，不過CORBA 提供新的解決方
案：以CORBA 為溝通介面的主從式程式設計架構，讓開發者在擴充開
放原始碼軟體新功能的同時，也可以保持程式碼的私有性。

------

開放原始碼運動對商業發展而言，程式開發者可以將成本投注於擴充
現有功能，不需要浪費時間重新發明輪子。然而，多數開放原始碼授
權同時也要求開發者釋出修改後的程式碼，有些授權更聲明不得將這
些程式碼用於商業軟體上。因此讓開發者對於使用開放原始碼軟體做
為解決方案一事感到猶疑不決。

是否有可能在自己的產品中使用開放原始碼程式，卻不放棄自己的智
慧財產權呢？ CORBA提供了一種可能性，讓開發者可以擴充開放原始
碼軟體的功能，卻不需要真的修改到原始的程式碼─甚至可以保留自
己的程式碼。

The Common Object Request Broker Architecture ﹙CORBA﹚ 是一
套讓不同程式之間彼此溝通的協定，程式可以在不同的機器上，使用
不同的作業系統或是不同的程式語言。舉例而言，一隻在Linux 上跑
的Java程式可以藉著CORBA跟在Windows上跑的C++ 程式溝通，使用這
隻C++程式的功能，取得這個C++程式所聯結到的資料庫資料。

透過主從式的程式設計架構﹙client-server  model﹚，將程式分成
兩個部份，開放原始碼程式放在公開的伺服器端，私有程式放在客戶
端，雙邊使用CORBA 介面溝通。對受到開放原始碼授權限制的程式開
發者而言，採用CORBA-based 的程式設計策略讓開發者可以將不欲公
開的程式碼放在客戶端，而且不違反開放原始碼授權。

開放原始碼軟體是世界上最被廣泛應用並且寫得最好的軟體。然而，
因為授權的關係，在商業應用上容易受到限制。除了向律師請教各種
在法律上可能採取的做法外，採用CORBA-based 的程式設計策略，將
程式分為公開與私有兩個部份是另一個新途徑。

■ 要你好玩  Linux新酷Game登場！

梁天佐/編譯
2001/04/13

在遊戲軟體方面， Linux和其他工作型作業系統類似，可供選擇的質
量不多；不過Linux 是全能型的作業系統，有許多一般使用者，遊戲
軟體有其必要。專門移植Windows平台遊戲軟體至Linux環境的Loki公
司，最近又推出兩款新遊戲：「阿爾發文明」與「星海生死鬥2」。

------

Loki Entertainment Software日前宣布，於4月16與18日陸續推出「
阿爾發文明」﹙Alpha Centauri﹚ 與「星海生死鬥2」﹙Tribes 2﹚
兩款原本在Windows平台上相當熱門的遊戲Linux版本。

「星海生死鬥2」是一款「毀滅戰士」型﹙Doom-like﹚的第一人稱動
作遊戲，玩家必須在令人屏息的遊戲空間中，運用各種新式花俏的武
器，有時還得駕駛各式前所未見的交通工具，在詭譎多變的天候下奮
力屠殺異族敵人，由於3D圖形炫麗精美，臨場的緊張與爽快感十足，
稱得上是第一人動作遊戲的優秀之作，值得在Linux平台上一試。

「阿爾發文明」則是與「三國志」類型相似，故事背景設定在太空世
紀的策略模擬冒險。當新世界的奇異太陽自地平線昇起，玩家必須思
考資源、環境、時間與敵對勢力等因素，努力壯大自己，侵略或消滅
敵國，玩家可以在遊戲中建立自己的理想國，以人類過往錯誤的歷史
為鑑，實現心目中的新世界。

執行「星海生死鬥2」需要Linux 2.2以上的穩定核心版本，由於需要
較高度3D處理，CPU最好是300MHz以上﹙建議500Mhz﹚， 至少4M的影
像卡並支援3D加速功能，XFree86 3.3.5以後的版本，64M記憶體﹙建
議128M﹚，硬碟空間300M﹙建議500M﹚，八倍速以上光碟機，及 OSS
十六位元相容音效卡。「阿爾發文明」需安裝glibc-2.1， 133MHz以
上的 CPU，記憶體16M以上，90M硬碟﹙建議200M﹚，四倍速以上光碟
機，其他大致相同。此外兩款遊戲都可進行連線對戰。

有鑒於Linux 遊戲軟體仍屬於未開發的市場，在一般使用者之間具有
潛力，Loki的經營方式為：與其他電腦遊戲公司合作，將知名遊戲移
植到Linux平台。目前已有十九款著名遊戲的Linux版上市；這種做法
不僅為遊戲公司以最少的成本開拓Linux市場，也為Linux使用者增加
了聲光效果豐富的遊戲選擇。

■ 擴展市場 微軟視窗媒體格式向Linux招手？

李孟諺/編譯
2001/04/13

微軟公司與製造嵌入式Linux串流媒體伺服器的StarBak公司合作，允
許該公司製造的 Torrent 100串流媒體網絡設備使用視窗媒體檔案格
式﹙Windows Media Format，WMF﹚。使用Linux作業平台將可使用視
窗媒體服務﹙Windows Media Services，WMS﹚。 微軟是否藉此拓展
串流媒體市場？

------

檔案格式是微軟維持產品市場佔有率的優勢之一。以微軟Office下的
Word為例， doc檔案格式讓其他文書軟體無法讀取微軟Word的檔案格
式。而且每當有新版推出時，舊版使用者必須升級，以便於讀取新的
檔案格式。

雖然特定的檔案格式讓微軟維持高市場佔有率，這樣的影響力對於開
放原始碼界來說不一定造成傷害。開放原始碼軟體有一些協定當初是
由微軟大力推廣，再經開放原始碼程式設計師改造後使用，例如DHCP
與SMP協定都是如此。

而現在微軟授權允許 StarBak公司將其視窗媒體檔案格式﹙ Windows
 Media Format，WMF ﹚應用在該公司製造的串流媒體伺服器，推廣
到Linux平台上。 StarBak是美國俄亥俄州專門製造可在嵌入式Linux
上使用的串流媒體伺服器的小公司，該公司總裁兼營運長Ray Harris
表示，該公司製造的Torrent 100 串流媒體網絡設備獲得微軟授權是
該公司的成功。

對於一家只有20人的小公司而言，這項合作是極大的成就。微軟數位
媒體行銷部經理Dave Fester 表示，與StarBak 這類創新的公司合作
開發支援Windows Media 的產品，讓信賴微軟的內容提供者有更多彈
性與選擇提供服務給他們的客戶。事實上，微軟在這項交易上不將作
業系統列為考量，而是注重自身串流媒體的地位，或許微軟想在串流
媒體市場上，從對手RealNetworks的手中搶下更多的地盤。

當StarBak 發表聲明的同一天，微軟也宣佈目前約有一千三百二十萬
的使用者採用視窗媒體檔案格式，以相較於RealNetWorks將近四倍的
成長速度增加，但是這數據遠遠被擁有兩千七百七十萬使用者的Real
Networks甩在後頭。因此，撇開作業系統不談，微軟想要的可能是更
多的使用者。

但從DHCP與SMP 協定的例子來看，微軟這項舉動或許是想要加強多媒
體市場的佔有率，不過對開放原始碼界來說，未必是件壞事。
